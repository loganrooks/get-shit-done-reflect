## What Happened

Plans 02 and 03 in phase 38.1 touched undeclared `.claude/` directory files beyond their declared scope. Plan 02 declared 8 files but commits also modified 8 undeclared `.claude/` installed copies. Plan 03 declared 2 files but also modified 1 undeclared `.claude/` file.

## Context

Phase 38.1 (project-local knowledge base), Plans 02 and 03. This project uses a dual-directory architecture where `agents/`, `get-shit-done/`, `commands/` are npm source directories that get copied to `.claude/` by the installer (`bin/install.js`). The undeclared files are all installer sync artifacts.

## Potential Cause

Plans do not declare `.claude/` installed copies in their `files_modified` frontmatter because these files are generated by the installer, not directly edited. However, the installer runs as part of plan execution (or is triggered by it), causing these files to appear in git commits. This is a recurring pattern -- see related signal from phase 35 (sig-2026-03-02-plan-scope-declaration-mismatch-35-03-and-04). Plans could either declare expected installer outputs or the scope comparison could exclude `.claude/` paths.
