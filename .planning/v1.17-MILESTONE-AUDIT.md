## Tech Debt

### Advisory Integration Findings (2 items)

1. **`auto_collect` config key without runtime consumer** (P40)
   - `signal_collection.auto_collect` is declared in feature-manifest.json but the auto_collect_signals postlude uses `resolve-level` exclusively for gating. The key is returned as a "knob" but never acted upon. Vestigial from earlier design iteration. Non-breaking.

2. **Hardcoded KB path in auto_reflect** (P42)
   - `execute-phase.md:593` reads `.planning/knowledge/index.md` without `~/.gsd/knowledge/` fallback. Protected by `|| echo 0` error handling. In practice, if project-local KB doesn't exist, signal collection would have been skipped earlier, so this path is unlikely to be reached. Inconsistent with P38.1 convention.

### Stale Documentation (1 item)

3. **Log sensor spec says "disabled by default"** (P38)
   - `gsd-log-sensor.md` body claims "disabled by default in manifest" but under the new extensible architecture all sensors default to enabled. Non-functional impact.

### Code Comments (1 item)

4. **Misleading comment on reflection-counter check** (P42)
   - `gsd-tools.js` check action writes config (initializes defaults) despite comment suggesting read-only. Consistent with other gsd-tools initialization patterns. Non-functional impact.
