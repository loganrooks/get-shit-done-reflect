Create `get-shit-done/bin/lib/telemetry.cjs` following the kb.cjs structural pattern. The module reads pre-computed session-meta JSON files from `~/.claude/usage-data/session-meta/` and optionally joins facets from `~/.claude/usage-data/facets/`.

**Module header** (mirrors kb.cjs):
```javascript
'use strict';
const fs = require('node:fs');
const path = require('node:path');
const os = require('node:os');
const { output, error, resolveWorktreeRoot, atomicWriteJson } = require('./core.cjs');
```

**Helper functions to implement** (all in the order listed):

1. `getSessionMetaDir()` — returns `path.join(os.homedir(), '.claude', 'usage-data', 'session-meta')`
2. `getFacetsDir()` — returns `path.join(os.homedir(), '.claude', 'usage-data', 'facets')`
3. `getTrustTier(session)` — deterministic exclusion rules per spk-007:
   - 'exclude' if `assistant_message_count === 0 && output_tokens === 0`
   - 'exclude' if `assistant_message_count <= 1 && output_tokens === 0 && first_prompt === 'No prompt'`
   - 'caveated' if `duration_minutes > 1000`
   - 'clean' otherwise
4. `computeDistribution(values)` — vanilla 12-line percentile: filter nulls/NaN, sort, compute {n, min, p25, median, p75, p90, max, mean} using linear interpolation. Return null for empty arrays.
5. `categorizeFirstPrompt(firstPrompt)` — lowercase before matching:
   - `/gsdr:` or `/gsd:execute-phase` → appropriate gsd_execute category
   - Recognize all standard GSD command prefixes: `/gsd:execute-phase` or `/gsdr:execute-phase` → `gsd_execute`; `/gsd:plan-phase` or `/gsdr:plan-phase` → `gsd_plan`; `/gsd:discuss-phase` or `/gsdr:discuss-phase` → `gsd_discuss`; `/gsd:research-phase` or `/gsdr:research-phase` → `gsd_research`; `/gsd:spike` or `/gsdr:spike` → `gsd_spike`; any other `/gsd` or `/gsdr` prefix → `gsd_other`; `null` or `'No prompt'` → `no_prompt`; length < 50 → `short_task`; ends with `?` → `question`; else → `freeform_task`
6. `computeHoursEntropy(messageHours)` — Shannon entropy from message_hours array. Returns 0 for empty/null.
7. `classifyFocusLevel(entropy)` — 'focused' (≤0.5), 'extended' (≤1.5), 'fragmented' (>1.5)
8. `loadSessionMetaCorpus(sessionMetaDir, opts)` — reads all `.json` files with try/catch (skip malformed; null-byte trim: `.trimEnd().replace(/\x00+$/, '')`). Applies trust-tier filtering. Applies project filter via `resolveWorktreeRoot()` on both sides. Returns array of session objects.
9. `loadFacetsIndex(facetsDir)` — loads all facets JSONs into `Map<session_id, facet>`, skipping malformed.
10. `enrichSession(session, facetsIndex)` — returns session with computed fields added: `_tier`, `_first_prompt_category`, `_hours_entropy`, `_focus_level`, `_facet` (null if not in facets).
11. `buildInterpretiveNotes()` — returns object with per-metric interpretive notes. Each note has `measures`, `does_not_measure`, `could_mislead`. Notes required for: `output_tokens`, `tool_errors`, `duration_minutes`, `user_interruptions`, `message_hours_entropy`, `first_prompt_category`, `facets_outcome`, `facets_friction`.

**Five subcommand functions:**

`cmdTelemetrySummary(cwd, opts, raw)`:
- Load corpus with project filter = cwd (clean tier only by default; opts.includeCaveated for caveated too)
- Enrich all sessions
- Compute corpus stats: total files scanned, clean count, caveated count, excluded count, skipped (malformed)
- Compute distributions for: output_tokens, tool_errors, duration_minutes, user_interruptions
- Compute first_prompt_category breakdown (counts per category)
- Compute focus_level breakdown (counts: focused/extended/fragmented)
- Facets coverage: count of sessions with _facet != null
- Include interpretive_notes
- Human-readable: summary table with key stats. JSON (--raw): full structured object.

`cmdTelemetrySession(cwd, sessionId, raw)`:
- Load corpus without project filter (session_id is globally unique)
- Find session matching sessionId (exact match on session.session_id or filename)
- Enrich it
- If not found: error('Session not found: <sessionId>')
- Output: all session-meta fields plus computed fields (_tier, _first_prompt_category, _hours_entropy, _focus_level)
- If _facet != null: include facets data with annotation `_facets_annotation: "AI-generated estimates with unknown accuracy (TEL-05)"`

`cmdTelemetryPhase(cwd, phaseNum, raw)`:
- Read STATE.md performance metrics section to find plan-level rows for the requested phase (e.g., phase "57" matches rows "57-01:", "57-02:", etc.)
- STATE.md perf format is `- {plan-id}: {N}min, {tasks} tasks, {files} files` — no timestamps available
- Fall back to phase directory creation time: check `.planning/phases/` for directory matching phase number pattern, use directory mtime as approximate start, current time as end
- Filter corpus sessions by: project_path matches cwd (with worktree normalization) AND start_time between phase start and end
- If no time window can be derived: return sessions filtered only by project_path, with a warning in output that phase-window correlation was unavailable
- Compute same distributions as summary but scoped to the phase window
- Include phase metadata: phase number, session count, time window used (or 'unavailable'), caveat if window approximated

`cmdTelemetryBaseline(cwd, opts, raw)`:
- Load full corpus with project filter = cwd (clean + caveated for completeness)
- Enrich all sessions
- Load facets index; compute facets metrics on matched subset (n=109 expected for this project)
- Compute distributions for ALL baseline dimensions:
  - output_tokens, tool_errors, duration_minutes, user_interruptions (primary behavioral)
  - first_prompt_category counts, focus_level counts (computed fields)
  - message_hours_entropy distribution
  - facets-derived (on matched subset only): friction_counts distribution, session_type breakdown, outcome breakdown
- Build baseline object:
  ```javascript
  {
    generated_at: new Date().toISOString(),
    schema_version: "1.0",
    runtime: "claude-code",
    project: resolveWorktreeRoot(cwd),
    corpus: { total_files, clean_count, caveated_count, excluded_count, malformed_count },
    facets_coverage: { total: N, matched: 109, coverage_pct: 40.7 },
    metrics: { /* output_tokens, tool_errors, etc. with distributions */ },
    computed_metrics: { /* first_prompt_category, focus_level, message_hours_entropy */ },
    facets_metrics: {
      _annotation: "AI-generated estimates with unknown accuracy (TEL-05)",
      _n: <facets matched count>,
      friction_counts: { distribution },
      session_type: { breakdown },
      outcome: { breakdown }
    },
    interpretive_notes: buildInterpretiveNotes(),
    token_validation: {
      primary_source: "output_tokens",
      rationale: "input_tokens is post-cache residual (1-3 tokens/call) — not a workload proxy. output_tokens validated within 0-8% against JSONL in spk-004.",
      input_tokens_warning: "input_tokens values in session-meta are NOT session token workload. They reflect cache-miss residuals only."
    }
  }
  ```
- Write to `path.join(resolveWorktreeRoot(cwd), '.planning', 'baseline.json')` using `atomicWriteJson()`
- Call `output()` with baseline object and raw flag

`cmdTelemetryEnrich(cwd, sessionId, raw)`:
- Same session lookup as cmdTelemetrySession
- Load facets index; find matching facet by session_id
- If no facet found: output session data with message "No facets data for this session"
- If found: merge session fields with facet fields, annotate ALL facets-derived fields with `_ai_estimate: true` and include `_facets_annotation: "AI-generated estimates with unknown accuracy (TEL-05)"`
- Output: merged object with explicit annotation

**Usage message** at bottom of routing (not in this task — handled by Task 2).

All subcommands: pass `raw` to `output()`. Use `error()` for failures (exits process).

DO NOT use `input_tokens` as a workload metric anywhere. Use `output_tokens` as the primary token metric per spk-004.

DO NOT mix clean and caveated sessions in default distributions without annotation. Default: clean only; add `includeCaveated` option for explicit opt-in.

```javascript
case 'telemetry': {
  const subcommand = args[1];
  const telemetryRaw = raw;
  if (subcommand === 'summary') {
    const includeCaveated = args.includes('--include-caveated');
    telemetry.cmdTelemetrySummary(cwd, { includeCaveated }, telemetryRaw);
  } else if (subcommand === 'session') {
    const sessionId = args[2];
    if (!sessionId) error('Usage: gsd-tools telemetry session <session-id>');
    telemetry.cmdTelemetrySession(cwd, sessionId, telemetryRaw);
  } else if (subcommand === 'phase') {
    const phaseNum = args[2];
    if (!phaseNum) error('Usage: gsd-tools telemetry phase <phase-number>');
    telemetry.cmdTelemetryPhase(cwd, phaseNum, telemetryRaw);
  } else if (subcommand === 'baseline') {
    const includeCaveated = args.includes('--include-caveated');
    telemetry.cmdTelemetryBaseline(cwd, { includeCaveated }, telemetryRaw);
  } else if (subcommand === 'enrich') {
    const sessionId = args[2];
    if (!sessionId) error('Usage: gsd-tools telemetry enrich <session-id>');
    telemetry.cmdTelemetryEnrich(cwd, sessionId, telemetryRaw);
  } else {
    error('Usage: gsd-tools telemetry <summary|session|phase|baseline|enrich>');
  }
  break;
}
```
