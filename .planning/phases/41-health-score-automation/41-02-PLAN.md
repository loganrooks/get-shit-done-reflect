Add a new top-level `health-probe` case to gsd-tools.js (alongside existing `automation` and `sensors` cases, before the `default` case). This subcommand handles complex health computations that are too involved for inline bash.

**CLI pattern:**
```
gsd-tools.js health-probe {probe-name} [--raw]
```

**Implementation: Add three functions and the case handler.**

**1. `cmdHealthProbeSignalMetrics(cwd, raw)` -- HEALTH-08:**
- Find KB directory: `.planning/knowledge/` primary, `~/.gsd/knowledge/` fallback
- Read `.planning/config.json` for `health_check.resolution_ratio_threshold` (default 5.0)
- Find latest regime_change entry: scan `signals/` dir for files containing `type: regime_change`, sort by created date, use latest as regime start. If none found, use epoch 0 (all history is one regime).
- Count signals within current regime by filtering signal files where created date >= regime start
- Separate detected (lifecycle: detected or triaged) vs resolved (lifecycle: remediated, verified, or closed)
- Compute ratio: detected.length / resolved.length (Infinity if resolved=0 and detected>0, 0 if both 0)
- Status: PASS if ratio <= threshold, WARNING if ratio > threshold
- Output JSON (when --raw):
```json
{
  "probe_id": "signal-metrics",
  "checks": [{
    "id": "SIG-RATIO-01",
    "description": "Signal-to-resolution ratio within current regime",
    "status": "PASS|WARNING",
    "detail": "Ratio N:1 (threshold: M:1)",
    "data": { "detected": N, "resolved": M, "ratio": R, "regime": "regime-id-or-null" }
  }],
  "dimension_contribution": {
    "type": "workflow",
    "signals": { "critical": C, "notable": N, "minor": M }
  }
}
```
- Human-readable output (without --raw): formatted text summary

**2. `cmdHealthProbeSignalDensity(cwd, raw)` -- HEALTH-09:**
- Same KB and regime resolution as signal-metrics
- Group regime signals by phase (parse phase from file path or frontmatter)
- Compute per-phase signal counts (densities array)
- Determine trend: if last 3 phases show increasing counts = "increasing", decreasing = "decreasing", else "stable". If fewer than 3 phases, always "stable".
- Status: PASS if stable or decreasing, WARNING if increasing
- Output JSON format similar to signal-metrics with probe_id "signal-density"

**3. `cmdHealthProbeAutomationWatchdog(cwd, raw)` -- HEALTH-07:**
- Read `.planning/config.json` for automation.stats (contains per-feature last_triggered timestamps and fires counts)
- Read feature-manifest.json for expected features (health_check, signal_collection, etc.)
- For each feature with a configured frequency:
  - Derive expected cadence from frequency: "every-phase" = check if last_triggered was within recent phase activity, "on-resume" = check if last_triggered was within 24h, "milestone-only" = relaxed check
  - If last_triggered is missing or stale beyond 3x expected cadence, flag as WARNING
- Status: PASS if all features healthy, WARNING if any stale
- Output JSON with probe_id "automation-watchdog", dimension_contribution type "infrastructure"

**Case handler (add before default case in main switch):**
```javascript
case 'health-probe': {
  const probeName = args[1];
  if (probeName === 'signal-metrics') {
    cmdHealthProbeSignalMetrics(cwd, raw);
  } else if (probeName === 'signal-density') {
    cmdHealthProbeSignalDensity(cwd, raw);
  } else if (probeName === 'automation-watchdog') {
    cmdHealthProbeAutomationWatchdog(cwd, raw);
  } else {
    error('Unknown health-probe. Available: signal-metrics, signal-density, automation-watchdog');
  }
  break;
}
```

<step name="discover_probes">
  Scan get-shit-done/references/health-probes/*.md (use path relative to GSD install dir).
  For each .md file:
  1. Parse YAML frontmatter to extract probe_id, category, tier, dimension, execution, depends_on
  2. Skip files that fail to parse (report as WARNING finding, never crash -- Pitfall 1 from research)
  3. Filter by tier: if --full flag NOT set, exclude probes with tier: full
  4. Filter by --focus: if --focus kb, only include probes matching category containing "KB"; if --focus planning, only include probes matching category containing "Planning"

  6. Topological sort (Kahn's algorithm): probes with no dependencies first, then dependents
  7. If circular dependency detected, report as FAIL finding and skip the cycle (Pitfall 2)
</step>

<step name="execute_probes">
  For each probe in topological order:

  Check if any depends_on probe FAILed -- if so, skip this probe with status "SKIPPED (dependency failed)".

  Execute based on execution type:

  **inline:** Read the probe file body, find bash code blocks under ## Checks section. For each check:
  - Run the bash code block via Bash tool
  - Parse output for PASS, WARNING, or FAIL
  - Handle intra-probe blocks: if a check's **blocks:** list includes subsequent check IDs, and this check FAILed, skip the blocked checks
  - Record result: {id, category, description, status, detail, repairable}

  **subcommand:** Run the declared gsd-tools.js command:
  ```bash
  node ~/.claude/get-shit-done/bin/gsd-tools.js health-probe {probe_id} --raw
  ```
  Parse the returned JSON. Each entry in "checks" array becomes a finding.

  **agent:** (Used by rogue-context.md in Plan 04) Spawn a subagent with the probe file as spec. Parse SENSOR OUTPUT delimiters for structured results. Not needed until Plan 04 adds rogue-context.md.
</step>

<step name="compute_score">
  After all probes execute, compute the two-dimensional health score per health-scoring.md:

  1. Infrastructure: aggregate all findings from dimension: infrastructure probes
     - HEALTHY: zero FAILs, zero WARNINGs
     - DEGRADED: at least one WARNING, zero FAILs
     - UNHEALTHY: any FAIL

  2. Workflow: aggregate findings from dimension: workflow probes
     - Use dimension_contribution from subcommand JSON output
     - Sum weighted signals (critical*1.0 + notable*0.3 + minor*0.1) after pattern dedup
     - Compare against workflow_thresholds (from config or defaults low=2.0, high=5.0)
     - LOW / MED / HIGH

  3. Composite: apply 3x3 matrix from health-scoring.md → GREEN / YELLOW / RED

  4. Write cache to ~/.claude/cache/gsd-health-score.json:
     { infrastructure, workflow, composite, weighted_sum, signal_count, resolution_ratio, density_trend, checked: unix_timestamp, phase, regime_id }
</step>
