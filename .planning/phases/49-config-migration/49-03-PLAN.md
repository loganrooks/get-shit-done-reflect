**C3: Worktree-aware resolution (upstream commit 0afffb1, released v1.27.0)**

Add `resolveWorktreeRoot` function to core.cjs (in the upstream section, before fork exports):

```javascript
/**
 * Detect if cwd is inside a linked git worktree and resolve to the main worktree root.
 * Returns cwd unchanged if not in a worktree or if cwd IS the main worktree.
 * @param {string} cwd
 * @returns {string}
 */
function resolveWorktreeRoot(cwd) {
  try {
    const gitDir = execSync('git rev-parse --git-dir 2>/dev/null', { cwd, encoding: 'utf-8' }).trim();
    // If git-dir contains "worktrees/", this is a linked worktree
    if (gitDir.includes('/worktrees/') || gitDir.includes('\\worktrees\\')) {
      const commonDir = execSync('git rev-parse --git-common-dir 2>/dev/null', { cwd, encoding: 'utf-8' }).trim();
      // commonDir is the .git dir of the main worktree; its parent is the main worktree root
      return path.resolve(cwd, path.dirname(commonDir));
    }
  } catch (e) {
    // Not a git repo or git not available -- return cwd as-is
  }
  return cwd;
}
```

Add `resolveWorktreeRoot` to the upstream module.exports block.

**Note about `withPlanningLock`:** The upstream commit also includes a `withPlanningLock()` function for file-based locking. This is a more complex function that is better deferred to Phase 50 (testing) or Phase 52 (feature adoption). For this plan, only adopt `resolveWorktreeRoot` which is the pure detection function.
