  <action>Create `get-shit-done/bin/lib/kb-transition.cjs` exporting `cmdKbTransition(cwd, signalId, newState, options, raw)`. Follow research Pattern 2 verbatim — the dual-write ordering is load-bearing and not negotiable:

1. Open kb.db connection, find the signal's .md file via `discoverSignalFiles(kbDir) + basename === '${signalId}.md'`.
2. Read file content with `fs.readFileSync(sigFile, 'utf-8')`, parse via `extractFrontmatter()`.
3. Call `assertLegalTransition(fm.lifecycle_state || 'detected', newState, strictness)` — helper imported from a new local block or colocated. `strictness` defaults to the `lifecycle_strictness` frontmatter field if present, else `'flexible'`. Legal transitions encoded from `agents/knowledge-store.md:213-225` state machine:
   - detected → triaged | blocked | invalidated
   - triaged → blocked | remediated | invalidated
   - blocked → triaged | remediated | invalidated
   - remediated → verified | detected (recurrence)
   - verified → detected (regression)
   - invalidated → (terminal)
   Under `strict` strictness, reject anything not in this table. Under `flexible`, allow but log a warning. Under `minimal`, allow everything.
4. Build the new frontmatter: `newFm = { ...fm, lifecycle_state: newState, updated: new Date().toISOString() }`; append to `lifecycle_log` the structured entry `{ event: newState, timestamp: <iso>, reason: options.reason, resolved_by_plan: options.resolved_by_plan }` (the latter two are optional).
5. Create `.bak` sidecar: `fs.copyFileSync(sigFile, sigFile + '.bak')` BEFORE any write.
6. `db.exec('BEGIN IMMEDIATE')`. Write file with `spliceFrontmatter()` result. `db.prepare('UPDATE signals SET lifecycle_state=?, updated=?, lifecycle_log=? WHERE id=?').run(newState, updated, JSON.stringify(newFm.lifecycle_log), signalId)` — if the schema does not have a `lifecycle_log` column today, don't add it; store log as a JSON-serialized field in `provenance_status` or as a new column — check kb.cjs schema for existing patterns. Keep SQL side idempotent wrt file side. `db.exec('COMMIT')`.
7. On any error inside the try block: `db.exec('ROLLBACK')`, restore file from `.bak`, rethrow. On success: delete `.bak`.

Output: text form `Transitioned sig-X: detected → remediated (reason: completed by 59-04-PLAN.md)`. Under `--raw`: JSON `{ signalId, from, to, reason, resolved_by_plan, timestamp }`.

Usage: `gsd-tools kb transition <signal-id> <new-state> [--reason <text>] [--resolved-by-plan <id>] [--strictness strict|flexible|minimal]`.
