**C7: Hook field validation (upstream commit c229259)**

Add `validateHookFields(settings)` function near the existing `cleanupOrphanedHooks()` function (after line ~1620). Implementation:
- If `!settings.hooks || typeof settings.hooks !== 'object'`, return settings unchanged
- Two-pass approach (no mutation during iteration):
  - Pass 1: For each event type key in settings.hooks, filter the hooks array. Valid entries must have: (a) a `hooks` property that is an array with at least one element, AND (b) each hook in the array must have either a `prompt` field (agent hook) or a `command` field (command hook). Remove entries that fail validation.
  - Pass 2: Delete event type keys whose arrays are now empty. If all event types deleted, delete settings.hooks entirely.
- Return the mutated settings object
- Log each removed entry: `  + Removed invalid hook entry from {eventType}`

- In the hook registration section (around line 2765 where `writeSettings(settingsPath, settings)` is called) -- call `validateHookFields(settings)` before that writeSettings call

**C6: resolve_model_ids for non-Claude runtimes (upstream commit 02254db)**

For non-Claude runtimes (OpenCode, Gemini, Codex), write `resolve_model_ids: "omit"` to `~/.gsd/defaults.json` during install. This tells the upstream init module's model resolution to skip alias expansion for runtimes that handle model IDs differently.

Add this logic after the existing `getGsdHome()` call in the install function. The `getGsdHome()` function returns the `~/.gsd/` directory path. After ensuring the directory exists:
```javascript
// C6: Set resolve_model_ids for non-Claude runtimes
if (!isClaude) {
  const gsdHome = getGsdHome();
  const defaultsPath = path.join(gsdHome, 'defaults.json');
  let defaults = {};
  if (fs.existsSync(defaultsPath)) {
    try { defaults = JSON.parse(fs.readFileSync(defaultsPath, 'utf8')); } catch {}
  }
  if (defaults.resolve_model_ids !== 'omit') {
    defaults.resolve_model_ids = 'omit';
    fs.writeFileSync(defaultsPath, JSON.stringify(defaults, null, 2) + '\n');
    console.log(`  ${green}+${reset} Set resolve_model_ids: omit for ${runtimeLabel}`);
  }
}
```

Where `isClaude` is `!isOpencode && !isGemini && !isCodex` (the default runtime). Place this after the hook registration section but before the final success message. This is complementary to the fork's MODEL_PROFILES in core.cjs (per DC-8 and G5 guardrails) -- the installer defaults.json affects the upstream init module's resolution path, while the fork's core.cjs resolveModelInternal handles runtime model profile lookup on a separate code path.
