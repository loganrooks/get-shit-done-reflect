**Part A: `quick.md` pre-commit integration**

Find the existing structure around line 163-173 (the `branch_name` / detection area per CONTEXT). Replace or augment the current logic with:

```bash
# GATE-03: classify staged files
if git diff --cached --name-only > /tmp/gate03-files.txt && [ -s /tmp/gate03-files.txt ]; then
  CLASSIFICATION_JSON=$(node $HOME/.claude/get-shit-done-reflect/bin/gsd-tools.cjs quick classify --files $(cat /tmp/gate03-files.txt) 2>&1)
  CLASSIFICATION_EXIT=$?

  if [ $CLASSIFICATION_EXIT -ne 0 ]; then
    echo "::notice::gate_fired=GATE-03 result=block classification=$(echo "$CLASSIFICATION_JSON" | jq -r .classification)"
    echo "GATE-03 blocks direct-to-main: staged files include runtime-facing or planning-authority paths."
    echo "$CLASSIFICATION_JSON" | jq .
    echo ""
    echo "Required: create a branch and PR for these changes. Run:"
    echo "  git checkout -b gsd/quick-$(date +%y%m%d-%N | tail -c 6)"
    echo "  git add <files>"
    echo "  git commit"
    echo "  gh pr create"
    exit 1
  fi

  echo "::notice::gate_fired=GATE-03 result=pass classification=pure-docs"
fi
```

The step must run BEFORE any `git commit` or `git push` step in the quick workflow, and it must be an exit-coded gate (not an advisory prompt).

**Part B: CI post-commit backstop on push to main**

Add a new job (or step within existing job) that runs only on `push` to main (not on pull_request — PRs don't bypass if GATE-01 / GATE-14 work):

```yaml
post_commit_gate_03:
  name: GATE-03 post-commit backstop
  if: github.event_name == 'push' && github.ref == 'refs/heads/main'
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
      with:
        fetch-depth: 2
    - uses: actions/setup-node@v4
      with:
        node-version: 22
    - run: npm ci --omit=dev
    - name: Classify last commit's files
      run: |
        FILES=$(git diff --name-only HEAD^ HEAD)
        if [ -z "$FILES" ]; then
          echo "::notice::gate_fired=GATE-03 result=pass reason=no_files_changed"
          exit 0
        fi
        CLASS=$(node get-shit-done/bin/gsd-tools.cjs quick classify --files $FILES 2>&1 || true)
        EXIT=$?
        if [ $EXIT -ne 0 ]; then
          # Check if this commit came through a PR (merge commit)
          MERGE=$(git log -1 --format=%P HEAD | wc -w)
          if [ "$MERGE" -ge 2 ]; then
            echo "::notice::gate_fired=GATE-03 result=pass reason=merge_commit_from_pr"
            exit 0
          fi
          echo "::notice::gate_fired=GATE-03 result=post_commit_block"
          echo "::error::GATE-03 post-commit backstop: direct-to-main commit contains runtime-facing or planning-authority files"
          echo "$CLASS" | jq . || echo "$CLASS"
          exit 1
        fi
        echo "::notice::gate_fired=GATE-03 result=pass"
```

The `MERGE=$(git log -1 --format=%P HEAD | wc -w)` check is CRITICAL: a 2-parent commit indicates merge-from-PR (where GATE-01 already gated CI). Only single-parent direct commits to main should fail this step.

**Placement:** the new CI job sits alongside the existing test + gate-01 + gate-02 / gate-13 steps. If the existing `test` job runs on both `pull_request` and `push`, the `post_commit_gate_03` job should be a SEPARATE job with its own `if:` guard so pull_request events skip it (they're already gated).
