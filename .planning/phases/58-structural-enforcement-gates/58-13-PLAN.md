  <action>
Create a new library `get-shit-done/bin/lib/reconcile.cjs` that composes existing state / roadmap primitives into a reconciliation orchestrator.

**Function contract:**

```javascript
// reconcilePhase(phaseNumber, options)
// options: { dryRun: bool, autoCommit: bool }
// Returns: { status: 'reconciled'|'block', changes: [...], unreconciled: [...] }
reconcilePhase(phaseNumber, options = {})
```

**Algorithm:**

1. **Snapshot current state:** call `cmdStateSnapshot` from `state.cjs` to get structured STATE.md content. Parse ROADMAP.md (via `roadmap.cjs` helpers if present — otherwise small inline parse) to find the current phase row. List phase plan checkboxes (`.planning/phases/NN-*/NN-*-PLAN.md` frontmatter + recent SUMMARY existence).

2. **Compute expected state:** for the named phase N:
   - Expected plan checkboxes: for each `NN-XX-PLAN.md`, checked if corresponding `NN-XX-SUMMARY.md` exists (or, more reliably, if a commit with `docs(NN-XX):` or `feat(NN-XX):` lands in the phase branch). Use the simpler signal first; refine if false positives.
   - Expected phase-row status: `complete` if all plans are checked AND VERIFICATION.md exists with a pass verdict; `in_progress` otherwise.
   - Expected STATE.md `stopped_at`: the latest activity timestamp from phase directory contents or last commit on phase branch.
   - Expected touched planning-authority files: any `.planning/*.md` file modified in the phase branch (git diff --name-only against main).

3. **Diff current vs expected:** compute changes per field. Accumulate a list `changes` of each update needed.

4. **Apply changes (unless dry-run):**
   - STATE.md updates via `cmdStateRecordSession` or a similar existing primitive (wrap in atomicWriteFileSync per Phase 55.1 precedent).
   - ROADMAP.md phase row update via `roadmap.cjs` helper; if no helper, do a minimal in-place YAML/markdown edit.
   - Plan checkboxes in the phase directory's master list (if one exists) or in ROADMAP.md plan bullets.

5. **Commit atomically (if autoCommit=true):** stage all modified files + invoke existing `commit` primitive with a scaffolded message "docs(NN): reconcile phase closeout — STATE.md, ROADMAP.md, N plan checkboxes". If any update fails, abort and restore original files (use `git stash` as rollback).

6. **Emit fire-event:** after success, `console.log("::notice title=GATE-10::gate_fired=GATE-10 result=reconciled fields=" + changes.length)`. On block (any unreconciled field with validation error), `console.log("::notice...result=block")` and exit non-zero.

**Unreconciled case:** if some fields can't be auto-reconciled (e.g., PLAN exists but no SUMMARY AND no VERIFICATION — ambiguous status), list them in `unreconciled` and exit with code 5 (distinct). Caller prints the list so user can manually resolve.

**CLI subcommand:** `gsd-tools phase reconcile <phase_number> [--dry-run] [--auto-commit] [--auto]`. Register in `gsd-tools.cjs`. Help text explains the reconciliation scope.

**Composition over replacement:** do NOT reimplement state-update logic. Call existing primitives (cmdStateRecordSession, cmdStateRecordMetric, roadmap helpers). The *orchestration* is new; the underlying writes are existing code paths.
