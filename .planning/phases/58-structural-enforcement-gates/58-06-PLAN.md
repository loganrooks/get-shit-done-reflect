**Part A: CI emission marker**

Add a final step to the existing CI job in `.github/workflows/ci.yml` that emits the GATE-01 fire-event marker:

```yaml
- name: GATE-01 emission marker
  if: always()
  run: |
    if [ "${{ job.status }}" = "success" ]; then
      echo "::notice title=GATE-01::gate_fired=GATE-01 result=pass phase=${{ github.base_ref || github.ref_name }}"
    else
      echo "::notice title=GATE-01::gate_fired=GATE-01 result=block phase=${{ github.base_ref || github.ref_name }}"
      echo "::error::GATE-01 blocks phase advancement — one or more required checks failed"
      exit 1
    fi
```

Place this AFTER the existing test + install-verify steps so it observes their outcomes. The `if: always()` ensures the step runs even when prior steps failed (we want the emission on failure too).

**Part B: `gsd-tools phase advance --require-ci-green` subcommand**

In `get-shit-done/bin/lib/phase.cjs`, add a `requireCiGreen(options)` exported function that:

1. Reads the current branch via `git rev-parse --abbrev-ref HEAD`.
2. Finds the open PR for that branch via `gh pr view --json number,state,statusCheckRollup`.
3. If no PR exists: prints a user-facing message "GATE-01: No PR exists for current branch; create one via `gh pr create` before advancing." and exits non-zero (exit 2).
4. If PR exists: polls `gh pr checks <pr-number> --required --watch` with a reasonable timeout (e.g., 30 minutes); on success, emits `{"gate":"GATE-01","result":"pass","pr":<n>,"branch":<branch>}` on stdout. On failure: exits non-zero (exit 1) after printing which required check failed.
5. In `--auto` mode, skip the `--watch` and do a single poll; exit 3 if checks still running (caller retries).

In `get-shit-done/bin/gsd-tools.cjs`, register the new subcommand: `gsd-tools phase advance --require-ci-green`. Route to `phase.cjs` `requireCiGreen`. Follow the existing phase-subcommand routing pattern (grep for existing `cmdPhase*` handlers).

Add usage message line. Include in the `--help` output for `phase`.

**Part C: Wire `execute-phase.md` offer_next step to call the new subcommand**

In `get-shit-done/workflows/execute-phase.md` around line 778-801 (the `offer_next` step with advisory `[y/n]`), insert a pre-step that invokes the exit-coded CLI:

```bash
# GATE-01: Block advancement until CI passes

  echo "GATE-01 blocks advancement. Fix CI, then re-run this step."

fi
```

The existing `[y/n]` advisory path remains as the user-interaction layer AFTER the CLI check. The structural enforcement is the CLI exit code.
