Ship GATE-15 — source↔installed mirror parity as a CI check. After each install, every source file under `agents/`, `get-shit-done/`, `commands/` must match the expected installed file under `.claude/agents/`, `.claude/get-shit-done-reflect/`, `.claude/commands/` after applying the installer's path-transformation (`replacePathsInContent`). Divergence = FAIL.

Purpose: Dual-directory hazard from `CLAUDE.md:15-27` — edits to `.claude/` that never propagate back to source are the Phase 22 Agent Protocol pattern (23-day undetected drift). Research R1 found the installer's check today only verifies directory existence + VERSION match; it does NOT verify byte-identical-after-transform parity. This plan closes that gap.

Output: `scripts/verify-install-parity.js` that imports the installer's path logic + CI step that runs it after install.

Per-gate Codex behavior: GATE-15 `applies` on both runtimes per `58-05-codex-behavior-matrix.md` — CI runtime-neutral.

Fire-event: `::notice::gate_fired=GATE-15 result=block path=<file>` stdout marker on first divergence; `result=pass` otherwise.

Note on `dist/MANIFEST`: Research R1 confirmed `dist/MANIFEST` does NOT exist. The effective manifest is `package.json` `files` array + installer path-transformation. The parity script uses the installer's code as source of truth rather than parsing a static manifest.

Build the parity-verification script that CI invokes.

**Script signature:**

```
node scripts/verify-install-parity.js <INSTALL_DIR>
```

Where `<INSTALL_DIR>` is the HOME-equivalent root where the installer placed files (CI sets up a tempdir and points installer at it).

**Implementation:**

0. Before starting, grep `bin/install.js` to confirm supported installer flags (checker M9 safeguard):
   ```
   grep -nE "^\s*const has[A-Z]|--claude|--opencode|--gemini|--codex|--all|--local" bin/install.js | head -20
   ```
   Verified at authoring time (2026-04-20): `bin/install.js:72` defines `const hasClaude = args.includes('--claude')`. The verification / CI invocations below use `--claude --local`; if that flag is ever removed from `install.js`, update the CI step to the then-correct invocation (likely `--local` alone with default runtime).

1. First, grep `bin/install.js` to find the named exports or the callable path-transformation function. Research R1 named `replacePathsInContent` at `bin/install.js:2868-2869`. If the function is defined but not exported, either:
   (a) extract it into a small helper module `bin/lib/install-paths.cjs` that `install.js` requires, and the parity script requires the same module, OR
   (b) if extraction is higher-risk than justified, use `require('../bin/install.js')` and grab the function via an export at the bottom of install.js that you add (minimal export — do NOT restructure install.js).

   Preference: option (a) for cleaner separation; fall back to (b) if extraction breaks tests.

2. Walk the source-tree paths declared in `package.json` `files` array (plus `agents/`, `commands/`, `get-shit-done/` which are the installer's copy roots):

```javascript
const SOURCE_ROOTS = ['agents', 'commands', 'get-shit-done', 'skills'];
const INSTALLED_ROOTS = {
  agents: '.claude/agents',
  commands: '.claude/commands',
  'get-shit-done': '.claude/get-shit-done-reflect',  // path-prefix substitution per install.js
  skills: '.claude/skills'
};
```

3. For each source file `<root>/<path>`:
   - Read source content.
   - Apply `replacePathsInContent(content)` to produce expected installed content.
   - Read actual installed content from `<INSTALL_DIR>/<INSTALLED_ROOT>/<path>`.
   - Compare byte-identical. On mismatch, report the first diff line and exit non-zero.

4. Expected transformations (per install.js current logic, grep to confirm; Research R1 noted `get-shit-done/` → `get-shit-done-reflect/` and `gsd-` → `gsdr-` in content):
   - Content prefix `get-shit-done/` → `get-shit-done-reflect/` (path references inside files)
   - Content prefix `gsd-` agent / skill names → `gsdr-` where applicable (the installer knows which — reuse)
   - `/gsd:` → `/gsdr:` (command references)
   - Directory path `get-shit-done/` at root → installed at `.claude/get-shit-done-reflect/`

5. On success: emit `::notice::gate_fired=GATE-15 result=pass total_files=<N>` and exit 0.
6. On first divergence: emit `::notice::gate_fired=GATE-15 result=block path=<path> reason=<why>` and exit 1.

**Ignorelist:** some files are legitimately divergent (e.g., if there are install-generated files like a README in the installed dir that the source doesn't have). Maintain an explicit ignorelist in the script header — if the installer generates any non-source files, list them here. Start empty; the script's first CI run will surface any real divergences.

**Tests:** create a small unit test in `tests/unit/verify-install-parity.test.js` that feeds a synthetic source+installed pair to the compare function and confirms pass/fail behavior. Use existing test framework (vitest per `npm test`).

Add a CI step that runs after the existing install-verify step and invokes `scripts/verify-install-parity.js`.

```yaml
- name: GATE-15 source/install parity
  run: |
    INSTALL_DIR=$(mktemp -d)
    HOME="$INSTALL_DIR" node bin/install.js --claude --local 2>&1
    if node scripts/verify-install-parity.js "$INSTALL_DIR"; then
      echo "::notice title=GATE-15::gate_fired=GATE-15 result=pass"
    else
      echo "::notice title=GATE-15::gate_fired=GATE-15 result=block"
      echo "::error::GATE-15: source/install parity violation (see script output above)"
      exit 1
    fi
```
