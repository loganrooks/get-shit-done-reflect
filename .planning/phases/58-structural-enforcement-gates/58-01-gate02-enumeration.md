# GATE-02 Enumeration

Authoritative enumeration of every merge-surface site in the source tree at the time
Phase 58 Plan 01 executed. Wave 2 Plan 07 reads this file to author the CI grep check.

## 1. Scope and Methodology

**In-scope search paths:**

- `agents/`
- `get-shit-done/`
- `commands/`
- `.claude/` (install-target mirror of `get-shit-done/` source)
- `skills/` (does not currently exist in the repo)
- `.codex/skills/` and `.codex/get-shit-done-reflect/` (Codex install mirror)
- `.github/workflows/`

**Excluded paths:**

- `.sonnet-run-archive/` — independence-preserving archive, not live surface
- `node_modules/`
- `.planning/` — planning artifacts reference invocations but do not invoke merges; hits there
  are references/citations (research notes, requirements, context) and not executable surfaces.

**Grep patterns:**

1. `"gh pr merge"` — PR-level merges via `gh` CLI.
2. `"git merge --squash"` — raw git squash surface (every hit non-conforming unless in a
   clearly marked `# historical` comment).
3. `"git merge --no-ff"` — conforming merge-with-history surface.

**Commands actually run (Grep tool equivalents of ripgrep):**

```
rg -n "gh pr merge" agents/ get-shit-done/ commands/ .claude/ .codex/ .github/
rg -n "git merge --squash" agents/ get-shit-done/ commands/ .claude/ .codex/ .github/
rg -n "git merge --no-ff" agents/ get-shit-done/ commands/ .claude/ .codex/ .github/
```

## 2. Per-Gate Codex Behavior

**GATE-02:** `applies` on both Claude Code AND Codex runtimes.

Rationale: The enforcement point is a CI grep check that runs on every PR touching the
in-scope paths. CI is runtime-neutral — the grep fires regardless of which runtime
(Claude Code or Codex) authored the commit. There is no per-runtime skip or variation;
this is a pure source-tree invariant check.

| Runtime | Applies? | Skip reason |
|---------|----------|-------------|
| Claude Code | applies | — |
| Codex CLI   | applies | — |

## 3. Fire-Event Mechanism

**Fire-event marker (emission point authored by Wave 2 Plan 07):**

```
::notice::gate_fired=GATE-02 result=block
```

The marker is emitted by the CI grep step in `.github/workflows/ci.yml` when any of
the following is true:

1. A `gh pr merge` invocation is found without an accompanying `--merge` flag
   (i.e. default, `--squash`, `--rebase`, or missing).
2. A `git merge --squash` invocation is found outside a `# historical` comment block.

Plan 07 wires the step and the emission. This plan (01) produces the substrate
(enumeration + squash-default fix in `complete-milestone.md`) that the step operates on.

### 5.3 Plan 07 CI grep scope recommendation

for invocation checks, matching the research run scope in `58-RESEARCH.md:741`. The

duplicating the GATE-02 check across mirrors.

to allow a one-off historical comment escape hatch (see `58-07-PLAN.md:97-101`).
