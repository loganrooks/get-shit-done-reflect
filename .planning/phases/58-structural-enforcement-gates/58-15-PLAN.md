**Library `get-shit-done/bin/lib/release.cjs`:**

```javascript
// checkReleaseBoundary({ sincePhaseCommit, lagThresholdDays = 7, auto = false })
// Returns: { status: 'current'|'lag'|'deferred', latest_tag: string, days_since_phase_merge: number, lag_doc_path?: string }
checkReleaseBoundary(options)
```

**Algorithm:**

1. Determine `sincePhaseCommit`: either passed explicitly or detected via the most recent commit on main matching `Merge pull request .* from gsd/phase-` (parse via `git log --first-parent main --grep='Merge pull request.*from gsd/phase-' -1 --format='%H %ct'`).

2. Get latest release tag: `git tag --list 'reflect-v*' --sort=-version:refname | head -1`.

3. Get tag date: `git log -1 --format=%ct <tag>`.

4. Compute `days_since_phase_merge = (now - phase_merge_time) / 86400`.

5. Compute `tag_behind_phase = tag_date < phase_merge_time` (true means tag is stale relative to phase merge).

6. If a file `.planning/release-lag.md` exists, parse its YAML frontmatter:
   - If `deferred_to` is a date in the future: return `{status: 'deferred', lag_doc_path}` (exit 2).
   - If `deferred_to` is in the past OR missing: treat as stale deferral → `{status: 'lag', lag_doc_path}` (exit 1, with note).

7. If no lag doc AND `tag_behind_phase == false` AND `days_since_phase_merge < lagThresholdDays`: `{status: 'current'}` (exit 0).

8. Otherwise: `{status: 'lag'}` (exit 1).

9. Emit fire-event on every invocation: `console.log("::notice title=GATE-11::gate_fired=GATE-11 result=" + status + " days=" + days_since_phase_merge)`.

**CLI:** `gsd-tools release check [--since <commit>] [--lag-threshold-days N] [--auto]`.

**Template `.planning/handoff/release-lag-template.md`:**

```markdown
~~~
doc_type: release_lag
schema_version: v1
lag_reason: "<short reason>"
deferred_to: "<ISO8601 date; when release is expected>"
deferred_at: "<ISO8601 when this doc was written>"
named_rationale: |
  <multi-line explanation why this phase merge is not releasing now —
   e.g., 'batching releases per v1.20 completion plan', or
   'CI broken on secret rotation; unblocking next rotation cycle'>
referenced_phase_merge: "<commit sha of phase merge that triggered this deferral>"
written_by:
  role: planner|executor|verifier|user
  at: "<ISO8601>"
  session_id: "<id>"
~~~
# Release Lag Deferral

<narrative body explaining the decision and unblock path>
```

```bash
# GATE-11: release-boundary assertion
RELEASE_OUT=$(node $HOME/.claude/get-shit-done-reflect/bin/gsd-tools.cjs release check --auto 2>&1)
RELEASE_EXIT=$?

case $RELEASE_EXIT in
  0)
    echo "GATE-11: release current — $RELEASE_OUT"
    ;;
  1)
    echo "GATE-11 BLOCK: release lag detected"
    echo "$RELEASE_OUT"
    echo ""
    echo "Options:"
    echo "  A) Fire release: gh workflow run publish.yml"
    echo "  B) Defer explicitly: cp .planning/handoff/release-lag-template.md .planning/release-lag.md && \$EDITOR .planning/release-lag.md"
    echo "  C) Both: defer now, schedule the release"
    echo "After choosing, re-run this step."
    exit 1
    ;;
  2)
    echo "GATE-11: release explicitly deferred — $RELEASE_OUT"
    ;;
  *)
    echo "GATE-11 error: $RELEASE_OUT"
    exit 1
    ;;
esac
```
