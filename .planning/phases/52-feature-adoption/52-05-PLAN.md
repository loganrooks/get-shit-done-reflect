**1. Register context-monitor hook in installer (ADT-01 wiring):**

The context-monitor is a PostToolUse hook (not SessionStart like existing hooks). The installer's current `ensureHook()` function works with `settings.hooks.SessionStart`. A new hook event type is needed.

In install.js, after the existing SessionStart hook registration block (around line 2897), add PostToolUse hook registration:

```javascript
// Configure PostToolUse hook for context monitoring
if (!settings.hooks.PostToolUse) {
  settings.hooks.PostToolUse = [];
}

// Context monitor hook - reads bridge file, injects context warnings
const contextMonitorCommand = buildLocalHookCommand(hookDir, prefix, 'gsdr-context-monitor');
ensureHook('gsdr-context-monitor', contextMonitorCommand, 'context monitor');
```

Wait -- the existing `ensureHook()` function searches `settings.hooks.SessionStart`. It needs to be generalized, OR a separate registration needs to happen for PostToolUse. Check the upstream installer to see how they handle PostToolUse hook registration. Reference:
```bash
git show upstream/main:bin/install.js | grep -A5 -B5 "PostToolUse\|context-monitor"
```

Follow upstream's pattern for PostToolUse registration. The key requirements:
- Hook must be registered under `settings.hooks.PostToolUse` (not SessionStart)
- Hook command uses `buildLocalHookCommand()` with the fork's hook prefix
- Label: 'context monitor'
