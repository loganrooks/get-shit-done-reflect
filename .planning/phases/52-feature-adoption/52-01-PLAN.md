**2. Context scaling fix (ADT-02):**
Replace the existing 80% scaling formula:
```javascript
// OLD: const used = Math.min(100, Math.round((rawUsed / 80) * 100));
// NEW:
const AUTO_COMPACT_BUFFER_PCT = 16.5;
const usableRemaining = Math.max(0, ((remaining - AUTO_COMPACT_BUFFER_PCT) / (100 - AUTO_COMPACT_BUFFER_PCT)) * 100);
const used = Math.max(0, Math.min(100, Math.round(100 - usableRemaining)));
```
Note: `remaining` is derived from the hook input data (100 - rawUsed). The fork's current code has `rawUsed` from `data.context_window?.used_percentage` or similar. Adjust variable names to match the fork's existing variable bindings.

**3. CLAUDE_CONFIG_DIR resolution (ADT-04):**

```javascript
const claudeDir = process.env.CLAUDE_CONFIG_DIR || path.join(homeDir, '.claude');
```

- `todosDir` -> `path.join(claudeDir, 'todos')`
- `cacheFile` (update check) -> `path.join(claudeDir, 'cache', 'gsd-update-check.json')`

Add a bridge file write block after the context metrics are computed. The bridge file is written on every Notification event so the context-monitor hook can read it:
```javascript

    remaining_percentage: remaining,
    used_pct: used,

  }));
```
