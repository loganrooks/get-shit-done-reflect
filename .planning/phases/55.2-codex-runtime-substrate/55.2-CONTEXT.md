### Runtime capability resolution (CODEX-01)

**Current state:** `automation.cjs` lines 114-130 — `--runtime` flag only recognizes `claude-code` and `full`. The heuristic fallback (lines 119-129) checks for `.claude/settings.json`; if missing, assumes constrained (no hooks, no task tool). Codex-cli gets incorrectly capped on every feature.

- [grounded] Add `codex-cli` as a recognized `--runtime` value alongside `claude-code` and `full`
- [grounded] Codex has `task_tool` — `codex features list` confirms `multi_agent stable true` (drift audit verified)
- [grounded] Heuristic fallback should also check for `.codex/config.toml` (or project-local `.codex/`) when `.claude/settings.json` is absent
- [grounded] Codex MCP support is confirmed (STDIO + Streamable HTTP)
- [working assumption] Codex hooks status is "under development" per `codex features list` — treat as conditionally available. Researcher should verify current hook shape and whether `config.toml` hook entries are functional or just structural placeholders
- [working assumption] Auto-detection via env vars (`CODEX_CLI` or similar) vs requiring explicit `--runtime` — researcher should determine if Codex sets any identifying env var during execution

### Agent and sensor dual-format discovery (CODEX-02)

**Current state:** `sensors.cjs` line 24 — sensor regex is `/^gsdr?-.*-sensor\.md$/` (`.md` only). Discovery checks `.claude/agents/` then `agents/`, never `.codex/agents/`. Same pattern in `cmdSensorsBlindSpots` (line 116).

- [grounded] Sensor discovery regex must add `.toml` format: match both `gsdr-*-sensor.md` and `gsdr-*-sensor.toml`
- [grounded] Discovery path must check `.codex/agents/` in addition to `.claude/agents/` and `agents/`
- [grounded] TOML frontmatter parsing differs from YAML — sensor metadata extraction (lines 34-49) needs a TOML path
- [grounded] `checkAgentsInstalled()` (if it exists in workflow text) needs the same dual-format treatment
- [working assumption] When both `.md` and `.toml` versions exist for the same sensor, deduplicate by sensor name (not by file) — researcher should verify if this ever occurs in practice (it shouldn't if install is clean, but stale installs could produce it)
- [working assumption] Discovery should merge results from all directories rather than short-circuit on first hit — the current "try .claude/ first, fall back to agents/" pattern would miss .codex/ entirely

### Brownfield detection exclusion (CODEX-01 adjacent)

**Current state:** `init.cjs` line 215 — `skipDirs` includes `.claude` but not `.codex`. A project-local `.codex/` directory with `.js` files inside (hook scripts, etc.) could trigger false-positive brownfield detection.

- [grounded] Add `.codex` to the `skipDirs` set in init.cjs — same treatment as `.claude`
- [grounded] Minimal change: one string addition to the Set constructor
