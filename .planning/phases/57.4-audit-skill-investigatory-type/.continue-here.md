<context>
This phase has TWO concerns that must be held together:

1. OPERATIONAL: Build the invocable /gsdr:audit skill that Phase 57.3 deferred. The command + agent pair, session directory creation, ground rule embedding, type-aware templates. This is concrete engineering.

2. EPISTEMIC: Develop the investigatory audit type and frame-reflexivity practice. The REVIEW.md (pre-phase-archive) is the critical artifact here — it argues that audit types smuggle assumptions about the form of failure, that the investigatory type should "hold diagnosis open" rather than closing on a failure form early, and that ALL audit types should include a frame-reflexivity closing step. This is not decoration — it comes from a real investigation where the existing taxonomy had no type for "what happened and how did we get here?"

The discuss-phase MUST ingest:
- pre-phase-archive/REVIEW.md — the philosophical deconstruction. Take its advice seriously. It subjects its own proposals to the same critique it applies to the existing taxonomy.
- pre-phase-archive/260409-pz6-CONTEXT.md — operational design decisions (discovery phase, orchestrator pattern, ground rule copying, session directory conventions)
- Phase 57.2 artifacts — claim-types.md, context-checker pattern, DISCUSSION-LOG.md sidecar (the epistemic vocabulary this builds on)
- Phase 57.3 artifacts — audit-conventions.md, audit-ground-rules.md (the infrastructure this operationalizes)
- The signal and deliberation (derivation chain)

User direction (verbatim): "don't just naively add a type — investigate the situation in which the audit is demanded" and "we need a newfound critical eye towards our framing, our perhaps implicit assumptions that we need to expose if we are to render our activity legible, traceable, to help with diagnoses and reflections and self-improvement cycles."

The REVIEW.md proposes 4 investigatory-specific ground rules (I1-I4):
- I1: Start from the situation, not from a theory
- I2: Let the investigation guide artifact selection
- I3: Present competing explanations
- I4: Name the position of the investigation

And a frame-reflexivity step for ALL audit types (distinct from Rule 4 escape hatch): "Did the choice of audit type shape what you found? What would a differently-typed audit have looked for?"
