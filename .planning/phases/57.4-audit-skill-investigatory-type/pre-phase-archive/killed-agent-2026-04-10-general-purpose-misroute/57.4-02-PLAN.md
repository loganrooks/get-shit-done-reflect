**Part A — Add Section 3.5: The Composition Principle**

After the subject obligations / three new obligations / excess section, add a new subsection titled "3.5 The Composition Principle".

Write this as prose, not pseudocode. The directive from CONTEXT.md tone_guidance: "Name the tension, navigate situationally" is a hermeneutic directive. The section must preserve that character.

Lift the 4-step principle from RESEARCH.md §"Composition Principle" / deliberation lines 156–164:

> "Obligations from different axes compose into a flat list. When obligations tension against each other (e.g., I2 'follow the evidence freely' vs. S1 'run the test'), the auditor must:
> 1. Name the tension
> 2. Name what about the situation creates it
> 3. Show how they navigated it — responsive to both demands
> 4. The resolution emerges from engagement with the situation, not from a precedence rule
>
> This is a hermeneutic principle, not an algorithmic one: **'If you find yourself cleanly ignoring one obligation in favor of another, you've likely stopped engaging with the tension.'**"

Then write 2–3 paragraphs of prose naming:
- Why this is hermeneutic, not algorithmic: if there were a precedence rule, the framework would be flattening situations into a decision table. The deliberation rejects this explicitly.
- The red flag: an output that resolves every tension cleanly is suspect. An output that says "there were no tensions" is more suspect still — it's evidence the auditor didn't engage with the composition at all.
- The Q1 uncertainty: the composition principle is untested (per CONTEXT.md open question). First 2–3 investigatory audits should be reviewed for collapse-to-one-side behavior. Reference Q1 explicitly without resolving it.
- The task-spec guidance: the orchestrator (command) should anticipate tensions and name them in the task spec ("The following obligations may tension in your situation: ..."). This is not a precedence prescription — it is a heads-up so the auditor engages rather than resolves silently.

End the section with a named "How I Navigated Tensions" section requirement: every investigatory-orientation audit output must contain a section titled "How I Navigated Tensions" (or equivalent). If the section says "there were no tensions," flag that as Q1 data for review.

Do NOT include pseudocode, if-then logic, or a decision table. The entire point of the composition principle is that it isn't those things.
