## Decisions & Deviations

### Decisions made during execution

- **Rule 5 placed in Section 1, not Section 2.** The plan specified this explicitly, and REVIEW.md frames Rule 5 as a core rule that applies to every audit. Section 2 (Rule 4) is an escape hatch; Rule 5 is a core obligation. Keeping them in distinct sections preserves the distinction between "what the rules failed to anticipate" (Rule 4) and "was the rule set the right rule set" (Rule 5).
- **I1 phrasing: "Start from the discrepancy, not a theory."** REVIEW.md line 71 uses "Start from the situation, not from a theory." The three-axis deliberation line 127 uses "Start from the discrepancy, not a theory." Both are authoritative; the deliberation is the operational form and RESEARCH.md Orientation Obligations section also uses "discrepancy." Adopted the deliberation wording because the investigatory obligation is specifically occasioned by a discrepancy (something expected, something delivered, a gap), and "situation" is broader than what I1 is pointing at. The REVIEW.md wording is footnoted via authority-trace but not used verbatim.
- **Framework invisibility relationship paragraph added.** The plan required verbatim retrospective wording for the obligation itself (done, as a blockquote). I added a separate "Relationship to I4 and Rule 5" paragraph explaining that I4 is about the auditor's position, Rule 5 is about the classification, and framework invisibility is about the structural edges of what the audit can see — because without this distinction the three obligations feel redundant. This is a clarification, not a deviation — it preserves the verbatim retrospective wording intact.
- **Core Rules 4 and 5 listed in template snippet inside Section 4 code block.** The template numbering runs 1-5 for core rules (Rules 4 and 5 are labelled by name inside the numbered list so the escape-hatch / frame-reflexivity character is visible in the copied text). This keeps the composition explicit for the command and agent that will consume the template.

### Tension encounter during writing (per "What the Obligations Didn't Capture" spirit)

One small tension emerged while writing Rule 5's anti-performativity warning. The warning says "an empty Rule 5 is compliance theater" — but the plan's tone_guidance G-3 also warned against turning Rule 5 into a checklist. There is a real tension between "refuse empty answers" (which invites checklist-style enforcement) and "don't make this a checklist" (which invites emptiness as the permitted endpoint). I navigated this by making the specific grounding questions *concrete* (not generic) and by framing the refusal as "the specific questions demand a concrete consequence in the findings, not a yes/no" — so the warning catches empty answers without turning the rule into a box-tick. Whether this navigation actually survives contact with agents is an open question — it's exactly the kind of tension Q1 (composition principle untested) says the first investigatory audits will test.

Recording this here because the plan itself carries obligations and surprising findings during execution are worth capturing per the retrospective spirit, even though the formal "What the Obligations Didn't Capture" section lives in audit-conventions.md (Plan 01's scope), not here.
