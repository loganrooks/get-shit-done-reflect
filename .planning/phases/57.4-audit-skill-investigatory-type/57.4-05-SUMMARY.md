## WF-01 Partial-Delivery Note

WF-01 is intentionally kept as `[ ]` rather than `[x]`. The cross-model delegation *capability* has shipped as part of Phase 57.4 Plan 03 (`/gsdr:audit` command with `--trust-cross-model` opt-in), but the requirement is not considered complete until the Q2 spike validates dispatch reliability. The experimental flag is a deliberate signal that the mechanism is in trial, not ready to serve as a reliable default.

**Remaining work for full WF-01 sign-off:**

1. Q2 spike (candidate): design an experimental protocol for measuring cross-model dispatch reliability — environment setup failures, early exit rates, instruction degradation observed in downstream audit output
2. Run the spike on a representative sample of audit sessions (sample size from SPIKE-06 design)
3. If the measurement is good enough, remove `--trust-cross-model` gate and mark WF-01 `[x]`
4. If the measurement finds structural issues, record findings as signals and revise the dispatch approach before flipping WF-01

This follow-up is not yet scheduled as a Phase or Spike; it is a Q2 candidate flagged in the WF-01 text itself so downstream planning sees the commitment.
