<task type="auto">
  <name>Task 2: Implement the separate store, registry contract, and query envelope</name>
  <files>
    get-shit-done/bin/lib/measurement/registry.cjs
    get-shit-done/bin/lib/measurement/store.cjs
    get-shit-done/bin/lib/measurement/query.cjs

  <action>
Implement the core measurement substrate.

Requirements for the store:
- separate SQLite database path and schema from `kb.db`
- rebuildable from raw sources
- stores extractor output rows / query inputs, not coarse final reports as canonical truth
- carries source-family, runtime, and freshness/rebuild metadata

Requirements for the registry:
- family-aware composition so later plans can add `runtime`, `derived`, `gsdr`, and `codex` extractors without conflicting core edits
- each extractor entry declares: `name`, `source_family`, `raw_sources`, `runtimes`, `reliability_tier`, `features_produced`, `serves_loop`, and `distinguishes`

Requirements for the query layer:
- returns JSON-first response envelopes
- reserves fields for `question`, `scope`, `features`, `interpretations`, `distinguishing_features`, `anomaly_register`, `provenance`, `reliability`, `coverage`, and `freshness`
- treats feature availability and runtime symmetry as separate concerns

Do not attempt to finish full loop-specific interpretations here. This plan defines the stable contract and substrate only.
  </action>
  <verify>
    - `rg -n "source_family|reliability_tier|distinguishes" get-shit-done/bin/lib/measurement/registry.cjs`
    - `rg -n "coverage|freshness|provenance|interpretations" get-shit-done/bin/lib/measurement/query.cjs`
    - `rg -n "kb\\.db" get-shit-done/bin/lib/measurement/store.cjs` returns zero matches
    - structured rebuild metadata exists in the store/query layer for stale-or-unknown freshness states
  </verify>
  <done>
The measurement substrate exists as a separate rebuildable cache/store with a registry and query envelope that later plans can populate.
  </done>
</task>
