- The first draft implementation/tests used more verbose provenance wording (`confirmed_or_mixed`, pending counts). The final implementation was tightened to the plan’s stricter must-have wording so the contract stays stable and conservative.
- The initial report privacy line only exposed a count. It was upgraded to include the actual computed content-derived feature names because the plan required naming the privacy-relevant features, not just acknowledging their existence.
- Scope remained the planned 2-of-6 human-readable loop templates. The universal JSON contract now carries provenance on every interpretation, while `intervention_lifecycle`, `signal_quality`, `cross_session_patterns`, and `cross_runtime_comparison` remain intentionally deferred.
