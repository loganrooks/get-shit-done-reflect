2. REPLACE with an idempotent migration block:

```javascript
  // 57.7 MEAS-GSDR-06: drop signal_fts virtual table. Previously reserved for Phase 59
  // full-text search, never populated, external-content mode referenced nonexistent
  // title/body columns on signals. Ripgrep remains the search fallback per MEAS-GSDR-06
  // option (b). Re-entry path for future full-text search is contentless FTS5 rewrite
  // (option c), NOT option (a) which would mix free-text into the canonical row schema.
  // First drop any triggers referencing signal_fts (defensive; no triggers exist at
  // time of writing — verified 2026-04-17).
  const ftsTriggers = db.prepare(
    "SELECT name FROM sqlite_master WHERE type='trigger' AND sql LIKE '%signal_fts%'"
  ).all();
  for (const trigger of ftsTriggers) {
    db.exec(`DROP TRIGGER IF EXISTS ${trigger.name};`);
  }
  db.exec('DROP TABLE IF EXISTS signal_fts;');
```

Rationale:
- `DROP TABLE IF EXISTS` is idempotent — handles both fresh kb.db (no prior signal_fts) and migration of existing kb.db containing the reservation.
- The defensive trigger-scan addresses Pitfall 5 in RESEARCH.md (future trigger addition would break naive DROP).
- Comment explicitly cites MEAS-GSDR-06 and option (b) for traceability.
