# 57.7-05 Intervention-Point Calibration

## Goal

Determine whether the 57.7 intervention-point heuristic is strong enough to ship live, or whether it should register as `not_available`.

Acceptance criteria from `57.7-CONTEXT.md` / `57.7-RESEARCH.md`:

- Long-session fire rate `>= 5%`
- False-positive rate `< 30%`

Operational definition under test:

- User-side intervention after `>= 2` consecutive assistant turns
- Guarded by `isHumanTurnRecord()`
- Seed marker set from phase context
- Calibration refinement allowed by plan discretion if the corpus reveals a structural marker the seed list misses

## Method

### Corpus

- Matched Claude sessions available in this repo corpus: `113`
- Bucket availability:
  - Short (`<20` records): `53`
  - Medium (`20-100` records): `6`
  - Long (`>100` records): `54`

### Sample selection

- Deterministic 50-session sample, ordered by a stable hash of `session_id`
- Requested bucket targets:
  - Short: `>= 10`
  - Medium: `>= 15`
  - Long: `>= 15`
  - Planning-ish: `>= 5`
  - Execution-ish: `>= 5`
- Actual selected counts:
  - Short: `18`
  - Medium: `6`
  - Long: `26`
  - Planning-ish: `8`
  - Execution-ish: `7`

Note: the corpus only exposes `6` matched medium sessions, so the planned `15`-session medium target was unattainable. The calibration uses all available medium sessions and documents the skew honestly.

### Heuristic refinement

The seed marker list by itself produced `0` matches across the 50-session sample. Manual inspection of all user turns that followed `>=2` assistant turns showed why:

- `13/14` candidate turns in the full matched corpus were Claude's explicit placeholder text: `[Request interrupted by user]`
- `1/14` candidate turns was a genuine human redirect (`"can you write this down..."`)

That means the corpus does contain intervention structure, but it is usually surfaced as an explicit runtime marker rather than a natural-language opening word. Because 57.7 is structural, not semantic, the calibrated heuristic adds this exact placeholder as a valid intervention marker while leaving the original seed regex intact.

Final calibrated heuristic (`57.7-v1`):

- `text === "[Request interrupted by user]"`, OR
- seed regex match on the leading user text

The generic `"can you ..."` opener was deliberately **not** added. It appeared once, and broadening to plain `can you` would widen the heuristic semantically without evidence that the corpus needs it.

Review rule:

- `TP` when the record is the explicit Claude placeholder `[Request interrupted by user]`, because it denotes a user stop/interrupt event inside a multi-assistant span.
- `FP` when the marker appears but does not denote a real user interruption.

## Limitations

- This is self-graded calibration by the same executor who wrote the heuristic.
- The positive signal is dominated by a runtime placeholder, not natural-language redirection text.
- Medium-session coverage is structurally weak because only six matched medium sessions exist in the current corpus.
- Turn-normalized density remains low even though long-session session fire rate clears the gating threshold.

## Verdict

**VERDICT: PASS**

Rationale:

- Long-session fire rate in the 50-session sample: `19.23%` (`5 / 26`), above the `>= 5%` threshold
- False-positive rate after reviewing every flagged turn in the full matched corpus: `0.00%`, below the `< 30%` threshold

## Shipping consequence

The extractor should ship **live**, not `not_available`.

Implementation notes for Task 2:

- Preserve `content_contract: 'derived_features_only'`
- Store only counts, normalized positions, and record indices
- Do **not** store `sampled_matches`
- Tag the extractor for `pipeline_integrity` and `agent_performance`
- Record `heuristic_version: '57.7-v1'` in provenance

## If revisited later

Phase 60+ should revisit whether a second intervention heuristic is needed for genuine natural-language redirects that do not use the interrupt placeholder. The obvious candidate (`/^can you\\b/i`) was observed but intentionally rejected here because it would broaden the heuristic without enough evidence.
