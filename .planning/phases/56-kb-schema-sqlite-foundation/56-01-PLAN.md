**Node version guard (CRITICAL -- see RESEARCH.md Pitfall 7):**
Use lazy require pattern. Do NOT require node:sqlite at module top level. Instead, use a try/catch guard inside a helper function that only triggers when a kb command is actually invoked. This prevents gsd-tools.cjs from failing on Node <22.5.0 for non-KB commands. Pattern:

```javascript
let _DatabaseSync = null;
function getDbSync() {
  if (_DatabaseSync) return _DatabaseSync;
  try {
    _DatabaseSync = require('node:sqlite').DatabaseSync;
    return _DatabaseSync;
  } catch (e) {
    const nodeVersion = process.version;
    console.error(
      `Error: node:sqlite requires Node.js >= 22.5.0 (current: ${nodeVersion})\n` +
      'The KB commands need the built-in SQLite module.\n' +
      'Upgrade Node.js: https://nodejs.org/en/download\n' +
      'Or use nvm: nvm install 22 && nvm use 22'
    );
    process.exit(1);
  }
}
```

**Imports from existing modules:**
- `require('./frontmatter.cjs')` for `extractFrontmatter` and `reconstructFrontmatter`
- `require('./core.cjs')` for `output`, `error`, `safeReadFile`
- `require('node:fs')`, `require('node:path')`, `require('node:os')`, `require('node:crypto')`

**KB path resolution:**
```javascript
function getKbDir(cwd) {
  const localKb = path.join(cwd, '.planning', 'knowledge');
  if (fs.existsSync(localKb)) return localKb;
  const globalKb = path.join(process.env.GSD_HOME || path.join(os.homedir(), '.gsd'), 'knowledge');
  return globalKb;
}
function getDbPath(cwd) {
  return path.join(getKbDir(cwd), 'kb.db');
}
```

**Database initialization (openKbDb):**
- Use `new DatabaseSync(dbPath, { enableForeignKeyConstraints: true })`
- Set `PRAGMA journal_mode=WAL` and `PRAGMA busy_timeout=5000`
- Call `initSchema(db)` which creates all tables with `CREATE TABLE IF NOT EXISTS`

**Schema tables (from RESEARCH.md Pattern 2, verified working):**

1. `signals` -- all frontmatter fields as columns. Key columns: id (TEXT PK), file_path (TEXT NOT NULL), project (TEXT DEFAULT ''), severity (TEXT DEFAULT 'minor'), lifecycle_state (TEXT DEFAULT 'detected'), polarity (TEXT DEFAULT 'negative'), signal_category (TEXT DEFAULT 'negative'), disposition (TEXT DEFAULT ''), signal_type (TEXT DEFAULT ''), detection_method (TEXT DEFAULT ''), origin (TEXT DEFAULT ''), created (TEXT DEFAULT ''), updated (TEXT DEFAULT ''), phase (TEXT DEFAULT ''), plan (TEXT DEFAULT ''), runtime (TEXT DEFAULT ''), model (TEXT DEFAULT ''), gsd_version (TEXT DEFAULT ''), occurrence_count (INTEGER DEFAULT 1), durability (TEXT DEFAULT ''), confidence (TEXT DEFAULT 'medium'), status (TEXT DEFAULT 'active'), content_hash (TEXT NOT NULL)

2. `signal_tags` -- (signal_id TEXT REFERENCES signals(id), tag TEXT, PRIMARY KEY (signal_id, tag))

3. `signal_links` -- (source_id TEXT REFERENCES signals(id), target_id TEXT NOT NULL, link_type TEXT NOT NULL, PRIMARY KEY (source_id, target_id, link_type)). link_type values: qualified_by, superseded_by, related_to, recurrence_of

4. `spikes` -- (id TEXT PK, file_path TEXT NOT NULL, project TEXT DEFAULT '', hypothesis TEXT DEFAULT '', outcome TEXT DEFAULT '', created TEXT DEFAULT '', updated TEXT DEFAULT '', status TEXT DEFAULT 'active', content_hash TEXT NOT NULL)

5. `spike_tags` -- (spike_id TEXT REFERENCES spikes(id), tag TEXT, PRIMARY KEY (spike_id, tag))

6. `meta` -- (key TEXT PK, value TEXT). Store schema_version='1', last_rebuilt timestamp, signal_count, spike_count.

**Signal file discovery (discoverSignalFiles):**
Recursive walk of `{kbDir}/signals/` collecting all .md files. Also walk `{kbDir}/spikes/` for spike files. Handle both project-scoped subdirectories and root-level legacy files. Return array of absolute paths.

**Frontmatter-to-SQLite mapping (signalToRow):**
Implement the mapping from RESEARCH.md "Frontmatter to SQLite Column Mapping with Defaults" code example. CRITICAL normalization functions:

- `normalizeStatus(status)`: open/resolved/remediated/detected -> 'active'; archived -> 'archived'; default -> 'active'
- `normalizeLifecycleFromStatus(status)`: If no lifecycle_state, infer from status. remediated/resolved -> 'remediated'; default -> 'detected'
- `normalizeSignalType(fm)`: Legacy signals put signal_type values in the type field. Map observation->deviation, architecture->capability-gap, positive-pattern/positive_pattern->good-pattern, etc.
- `mapSourceToDetectionMethod(source)`: manual->'manual'; auto/automated/auto-collected->'automated'; deliberation-trigger->'manual'; SUMMARY/PLAN refs->'automated'; absent->'unknown'
- `mapSourceToOrigin(source)`: manual->'user-observation'; auto/automated/auto-collected->'collect-signals'; deliberation-trigger->'deliberation-trigger'; SUMMARY/PLAN refs->'plan-summary'; absent->'unknown'
- `normalizeSeverity(sev)`: Pass through as-is (the schema uses the original severity values: critical, notable, minor, trace, high, medium, low -- Phase 59 can normalize further)

**Content hashing:**
`require('node:crypto').createHash('sha256').update(content).digest('hex')` for each file.

**cmdKbRebuild(cwd, raw):**
1. Resolve kbDir via getKbDir(cwd)
2. Open/create db via openKbDb(getDbPath(cwd))
3. Discover signal files and spike files
4. Begin transaction
5. For each signal file: read content, compute hash, check existing hash (skip if unchanged), extractFrontmatter, signalToRow, INSERT OR REPLACE into signals + signal_tags + signal_links
6. For each spike file: similar process into spikes + spike_tags
7. Update meta table (last_rebuilt, signal_count, spike_count, schema_version)
8. Commit transaction
9. Output summary: `{total} files processed ({signals} signals, {spikes} spikes), {added} added, {updated} updated, {skipped} unchanged, {errors} errors`
10. If `raw` flag, output JSON: `{ signals: N, spikes: N, added: N, updated: N, skipped: N, errors: N, error_details: [...] }`
11. On parse errors: log warning per file but continue processing. Collect error details for reporting.

**Tags extraction:** Parse the `tags` field from frontmatter. It may be an array `[tag1, tag2]` or a string `"tag1, tag2"`. Use `Array.isArray(fm.tags) ? fm.tags : (typeof fm.tags === 'string' ? fm.tags.split(',').map(t => t.strip ? t.strip() : t.trim()) : [])`. INSERT each tag into signal_tags.

**Links extraction:** Parse `qualified_by` (array), `superseded_by` (string), `related_signals` (array), `recurrence_of` (string) from frontmatter. INSERT each into signal_links with appropriate link_type.

**cmdKbStats(cwd, raw):**
1. Open db (if not exists, say "No KB index found. Run `gsd-tools kb rebuild` first.")
2. Query counts: total signals, total spikes, by severity, by lifecycle_state, by polarity, by project, by signal_category, by detection_method
3. Query meta table for last_rebuilt, schema_version
4. Default output: human-readable table format with sections
5. If `raw` flag: JSON output with all counts

**cmdKbMigrate(cwd, raw):**
1. Resolve kbDir, discover signal files
2. For each file with a `source` field in frontmatter:
   a. Read file content
   b. extractFrontmatter
   c. Map source to detection_method and origin (using mapSourceToDetectionMethod/mapSourceToOrigin)
   d. Add detection_method and origin to frontmatter
   e. Remove old source field
   f. reconstructFrontmatter to rewrite the file
3. Track: files_modified, files_skipped (no source field or already migrated), errors
4. Output summary: `{modified} files migrated, {skipped} skipped, {errors} errors`
5. If `raw`: JSON output
6. Do NOT touch kb.db -- the user runs `kb rebuild` after migration to update the index

**Important constraints:**
- NEVER modify .md files in cmdKbRebuild (RESEARCH.md anti-pattern 1)
- All SQLite operations synchronous (DatabaseSync, not async -- RESEARCH.md anti-pattern 4)
- Use transaction wrapping for rebuild performance (RESEARCH.md Pitfall 5)
- Handle all 4 schema generations gracefully (RESEARCH.md Pitfall 1)
- Derive project from frontmatter `project` field, not directory structure (RESEARCH.md Pitfall 6)
