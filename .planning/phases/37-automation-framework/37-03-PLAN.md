```javascript
function cmdAutomationTrackEvent(cwd, feature, event, reason, raw) {
  if (!feature || !event) {
    error('Usage: automation track-event <feature> <fire|skip> [reason]');
  }
  if (event !== 'fire' && event !== 'skip') {
    error('Event must be "fire" or "skip"');
  }

  const configPath = path.join(cwd, '.planning', 'config.json');
  if (!fs.existsSync(configPath)) {
    error('No .planning/config.json found.');
  }

  // Normalize feature name
  const normalizedFeature = feature.replace(/-/g, '_');

  // Read-modify-write with atomic write
  const config = JSON.parse(fs.readFileSync(configPath, 'utf8'));
  if (!config.automation) config.automation = {};
  if (!config.automation.stats) config.automation.stats = {};
  if (!config.automation.stats[normalizedFeature]) {
    config.automation.stats[normalizedFeature] = {
      fires: 0,
      skips: 0,
      last_triggered: null,
      last_skip_reason: null,
    };
  }

  const stats = config.automation.stats[normalizedFeature];
  if (event === 'fire') {
    stats.fires++;
    stats.last_triggered = new Date().toISOString();
  } else if (event === 'skip') {
    stats.skips++;
    stats.last_skip_reason = reason || 'unknown';
  }

  // Atomic write: write to tmp, then rename
  const tmpPath = configPath + '.tmp';
  fs.writeFileSync(tmpPath, JSON.stringify(config, null, 2) + '\n');
  fs.renameSync(tmpPath, configPath);

  output({ feature: normalizedFeature, event, stats }, raw);
}
```

Add an automation level indicator to `hooks/gsd-statusline.js`. The indicator should display:
- `Auto:N` when configured level equals effective level (no runtime cap)
- `Auto:N(M)` when effective level is lower than configured (runtime-capped)

Insert the indicator logic AFTER the devTag section and BEFORE the output section. The indicator reads from `.planning/config.json` in the workspace directory.

```javascript
// Automation level indicator
let autoTag = '';
try {
  const autoConfigPath = path.join(dir, '.planning', 'config.json');
  if (fs.existsSync(autoConfigPath)) {
    const cfg = JSON.parse(fs.readFileSync(autoConfigPath, 'utf8'));
    if (cfg.automation && cfg.automation.level !== undefined) {
      const configured = cfg.automation.level;
      // Runtime cap heuristic: check if hooks are available
      // If .claude/settings.json has hooks configured, assume full capability
      let effective = configured;
      try {
        const settingsPath = path.join(dir, '.claude', 'settings.json');
        if (fs.existsSync(settingsPath)) {
          const settings = JSON.parse(fs.readFileSync(settingsPath, 'utf8'));
          if (!settings.hooks || Object.keys(settings.hooks).length === 0) {
            // No hooks configured -- cap features that need hooks
            // Simple heuristic: if any hook-dependent feature would be capped, show the cap
            if (configured > 2) effective = 2;
          }
        } else {
          // No settings.json -- assume hookless runtime
          if (configured > 2) effective = 2;
        }
      } catch {}

      if (effective < configured) {
        autoTag = `\x1b[36mAuto:${configured}(${effective})\x1b[0m \u2502 `;
      } else {
        autoTag = `\x1b[36mAuto:${configured}\x1b[0m \u2502 `;
      }
    }
  }
} catch {}
```

Design notes per research:
- The statusline shows CONFIGURED level and RUNTIME-CAPPED effective level only. It does NOT show context-deferred levels because the statusline hook process lacks access to the LLM's context window state. Context deferral is reported inline during workflow execution.
- The statusline hook must be fast (<50ms). File reads are synchronous and cached by the OS. No network calls, no subprocess spawns.
- The runtime cap heuristic in the statusline is simplified: if hooks are absent and configured > 2, cap at 2. This is a conservative approximation. The full resolution (with per-feature capability map) happens in gsd-tools.js resolve-level.
- Color: cyan (\x1b[36m) matches the existing statusline aesthetic.
