```markdown
# Measurement Diagnostics

This directory holds interpretation-object artifacts produced by running the measurement
substrate against real observed patterns. Each diagnostic follows a post-Popperian schema:
competing readings, distinguishing features (marked computed or uncomputed), anomaly register,
provenance block. Artifacts are source-controlled so Phase 57.7's interpretation-revision
classification has a target to classify (progressive vs degenerating per Lakatos).

## Artifact Schema

Every diagnostic is a markdown file with frontmatter + body:

### Frontmatter (YAML)
- `diagnostic_id`: `diag-YYYY-MM-DD-<slug>` (dated + slug)
- `subject`: short description of the observed pattern
- `produced_by`: command(s) + manual synthesis used to produce the diagnostic
- `observed_at`: ISO8601 timestamp of the measurement substrate rebuild used
- `measurement_store_rebuild_id`: hash or identifier from `store.cjs` rebuild
- `extractor_registry_size_at_diagnosis`: integer (for cross-referencing later)
- `loops_exercised`: array of loop keys (subset of LOOP_DEFINITIONS keys)

### Body Sections (required)

1. **Summary** — one paragraph prose framing the pattern.
2. **Evidence (from measurement substrate)** — bulleted list of features consulted, with
   which extractor produced them and what the value payload showed.
3. **Competing Readings** — at least 3 alternative interpretations of the evidence. Each reading has:
   - Name + source (which audit/signal/research surfaces it)
   - Evidence weight (strong/medium/weak)
   - Distinguishing feature: what WOULD distinguish this reading from the others, marked
     `(computed)` if 57.6 substrate already produces it, `(UNCOMPUTED — deferred to 57.7)`
     if a future extractor is needed.
4. **Anomaly Register** — any items from `queryResult.anomaly_register`, even if empty
   (empty anomaly register is data, not absence).
5. **Interpretation Revision Classification** — state that this is DEFERRED to 57.7
   (Lakatosian progressive/degenerating). Diagnostic documents the reading space.
6. **Provenance** — rebuild id, registry size, features consulted, report commands issued,
   explicit statement of what is UNCOMPUTED in current substrate.

## Index of Diagnostics

| Diagnostic | Subject | Primary Loops | Status |
|---|---|---|---|
| 2026-04-17-phase-57-vision-drop.md | Phase 57 vision-drop cascade | intervention_lifecycle, pipeline_integrity | committed (57.6-07) |
| 2026-04-17-facets-coverage-asymmetry.md | Facets coverage size-bias on agent performance | signal_quality, agent_performance | committed (57.6-07) |

## Non-Goals

- **NOT a dashboard**: diagnostics are prose + data excerpts, not live-updating views.
- **NOT behavioral-change measurement**: per RESEARCH.md Q3, 57.6 diagnostics do NOT require
  intervention-outcome tracking. That capability lands in 57.7 (MEAS-ARCH-09 deepening).
- **NOT success theater**: if the substrate cannot support a chosen reading's distinguishing
  feature, mark it UNCOMPUTED and document the gap. Do not tune the diagnostic to whatever
  renders cleanly (G-6).
```
