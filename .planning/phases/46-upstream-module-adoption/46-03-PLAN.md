    **Category 1: Upstream commands routed to modules**
    ```bash
    node get-shit-done/bin/gsd-tools.cjs state --raw
    node get-shit-done/bin/gsd-tools.cjs state get --raw
    node get-shit-done/bin/gsd-tools.cjs state-snapshot --raw
    node get-shit-done/bin/gsd-tools.cjs find-phase 46 --raw
    node get-shit-done/bin/gsd-tools.cjs generate-slug "test slug generation" --raw
    node get-shit-done/bin/gsd-tools.cjs current-timestamp date --raw
    node get-shit-done/bin/gsd-tools.cjs roadmap analyze --raw
    node get-shit-done/bin/gsd-tools.cjs validate consistency --raw
    node get-shit-done/bin/gsd-tools.cjs progress json --raw
    node get-shit-done/bin/gsd-tools.cjs phases list --raw
    ```

    **Category 2: Fork-specific commands routed to inline functions**
    ```bash
    node get-shit-done/bin/gsd-tools.cjs manifest validate --raw
    node get-shit-done/bin/gsd-tools.cjs backlog list --raw
    node get-shit-done/bin/gsd-tools.cjs automation resolve-level signal_collection --raw
    node get-shit-done/bin/gsd-tools.cjs sensors list --raw
    ```

    **Category 3: Fork init overrides (must use --include correctly)**
    ```bash
    node get-shit-done/bin/gsd-tools.cjs init execute-phase 46 --raw
    node get-shit-done/bin/gsd-tools.cjs init plan-phase 46 --raw
    node get-shit-done/bin/gsd-tools.cjs init progress --raw
    node get-shit-done/bin/gsd-tools.cjs init execute-phase 46 --include state,config --raw
    ```
    The last command must return JSON with `state_content` and `config_content` fields populated.

    **Category 4: Upstream init subcommands routed to init.cjs**
    ```bash
    node get-shit-done/bin/gsd-tools.cjs init quick "test description" --raw
    node get-shit-done/bin/gsd-tools.cjs init resume --raw
    ```

    **Category 5: New upstream commands (should work, not error)**
    ```bash
    node get-shit-done/bin/gsd-tools.cjs stats json --raw 2>&1 || true
    node get-shit-done/bin/gsd-tools.cjs validate health --raw 2>&1 || true
    ```
    These may not produce full output (e.g., stats may have different expectations), but they must NOT produce "Unknown command" errors.

    **Category 6: --cwd support (upstream improvement)**
    ```bash
    node get-shit-done/bin/gsd-tools.cjs --cwd "$(pwd)" state --raw
    node get-shit-done/bin/gsd-tools.cjs --cwd="$(pwd)" generate-slug "cwd test" --raw
    ```
    Both must succeed (same output as without --cwd since we pass current directory).

    For each command, verify:
    1. No crash (exit code 0, or expected error for missing args)
    2. JSON output is valid (parseable)
    3. No "ReferenceError" or "TypeError" in stderr (would indicate missing imports)

    Also verify the installed copy works:
    ```bash
    node .claude/get-shit-done-reflect/bin/gsd-tools.cjs state --raw
    node .claude/get-shit-done-reflect/bin/gsd-tools.cjs manifest validate --raw
    node .claude/get-shit-done-reflect/bin/gsd-tools.cjs init plan-phase 46 --raw
    ```
