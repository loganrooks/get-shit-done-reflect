Output: Fixed core.cjs functions, updated callers in phase.cjs and roadmap.cjs, 4+ regression tests.

Two independent sub-bugs in core.cjs, both must be fixed:

**Sub-bug A: Extend extractCurrentMilestone positive lookup (core.cjs:1150-1159)**

The current `sectionPattern` only matches markdown headings (`^#{1,3}\s+.*version`). Extend it to also match:
1. `<summary>` tags containing the version: `<summary>[^<\n]*VERSION[^<\n]*</summary>`
2. Bullet entries with the version: `^[-*]\s+[^\n]*VERSION`

When the match is a `<summary>` tag, the section boundaries must be the enclosing `<details>...</details>` block. Find the `<details>` open tag before the `<summary>` match and the `</details>` close tag after it.

When the match is a bullet entry, the section end is the next bullet entry with a version pattern or the next heading, or EOF.

The heading match path must continue to work exactly as before (regression safety).

**Sub-bug B: Rewrite replaceInCurrentMilestone to delegate to extractCurrentMilestone (core.cjs:1197-1206)**

Add an optional `cwd` parameter: `replaceInCurrentMilestone(content, pattern, replacement, cwd)`.

When `cwd` is provided:
1. Read STATE.md to get the current milestone version (same logic as extractCurrentMilestone lines 1126-1137)
2. Use the extended section pattern from Sub-bug A to find the current milestone section's start and end offsets in the raw content
3. Apply the replacement only within that range
4. Return the reconstructed content: `before + modified_section + after`

When `cwd` is NOT provided (backward compatibility): fall back to the original `lastIndexOf('</details>')` heuristic unchanged.

Do NOT duplicate the regex logic -- extract a helper function (e.g., `findCurrentMilestoneRange(content, cwd)`) that returns `{ start, end }` offsets, and have both `extractCurrentMilestone` and `replaceInCurrentMilestone` use it.

**Update callers:**

In `phase.cjs`, lines 715 and 744 call `replaceInCurrentMilestone(roadmapContent, pattern, replacement)`. Both are inside `cmdPhaseComplete` which has `cwd` as a parameter. Add `cwd` as the fourth argument to both calls.

In `roadmap.cjs`, lines 321 and 329 call `replaceInCurrentMilestone(roadmapContent, pattern, replacement)`. Both are inside `cmdRoadmapUpdatePlanProgress(cwd, ...)`. Add `cwd` as the fourth argument to both calls.

**Do NOT:**
- Remove `stripShippedMilestones` from exports (still used as a last-resort fallback inside `extractCurrentMilestone`)
- Modify `stripShippedMilestones` itself
- Change the behavior when `cwd` is not provided
