**In health-probe.cjs**, add a new function `cmdHealthProbeValidationCoverage(cwd, raw)` after `cmdHealthProbeAutomationWatchdog`. Follow the exact probe pattern (DC-4):

1. Scan `.planning/phases/` for directories.
2. For each directory, look for files ending in `-VALIDATION.md`.
3. Parse frontmatter to extract `compliance_pct` (regex: `/^compliance_pct:\s*(\d+)/m`).
4. Read threshold from `.planning/config.json` at `health_check.validation_threshold_pct` (default: 80 per Claude's discretion).
5. Compute: phases_scanned, phases_with_validation, average_compliance_pct, below_threshold list.
6. Status: PASS if no phases below threshold, WARNING otherwise.
7. Return shape:
```javascript
{
  probe_id: 'validation-coverage',
  checks: [{
    id: 'VAL-COVERAGE-01',
    description: 'Nyquist validation coverage across phases',
    status: status,
    detail: `${withValidation}/${scanned} phases validated, average compliance: ${avgCompliance}%`,
    data: { phases_scanned, phases_with_validation, average_compliance_pct, below_threshold }
  }],
  dimension_contribution: { type: 'quality', signals: { critical: 0, notable: belowThreshold.length > 0 ? 1 : 0, minor: 0 } }
}
```
8. Handle edge cases: no phases directory (PASS, "No phases directory found"), no VALIDATION.md files (PASS, "No validation files found").
9. If raw: `process.stdout.write(JSON.stringify(result)); process.exit(0);`
10. Otherwise: human-readable console output, then `process.exit(0)`.
