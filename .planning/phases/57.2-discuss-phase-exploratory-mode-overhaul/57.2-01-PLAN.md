**Structure the document with these sections:**

1. **Header** — Title, purpose statement ("Shared reference for claim type vocabulary in CONTEXT.md"), consuming agents listed (discuss-phase, researcher, context-checker).

2. **7 Claim Types table** — For each type (evidenced, decided, assumed, open, projected, stipulated, governing), include:
   - Definition (what it means)
   - Assignment criteria (when to assign this type)
   - Researcher action (what the researcher does with this type)
   - Justificatory expectation (what DISCUSSION-LOG.md must record for this type)

   Source for all 7 definitions: deliberation Analysis section "The convergence suggests 7 primary types" table. Source for justificatory expectations: deliberation "Justificatory expectations by type" table.

3. **Verification Dimension** — 3 levels (cited, reasoned, bare):
   - Definition of each level
   - Examples per level
   - Guidance: encourage `cited` where possible, flag `bare` claims that could be upgraded

4. **Notation Syntax** — Per RESEARCH.md Pattern 1:
   - Full form: `[decided:cited]`
   - Short form: `[assumed]` (defaults to bare)
   - Split claims: `[evidenced/assumed]`
   - Placement: start of claim line, after any label prefix
   - Regex for agent parsing: `\[(evidenced|decided|assumed|open|projected|stipulated|governing)(?:\/(evidenced|decided|assumed|open|projected|stipulated|governing))?(?::(cited|reasoned|bare))?\]`

5. **What Is NOT a Type** — Brief section explaining that phantom, invisible assumptions, scope/boundary, self-correcting, retroactive documentation are not types (per deliberation "What types are NOT in the ontology and why" section). This prevents agents from inventing extra types.

6. **Claim Dependencies** — Recording format guidance:
   - Use a table with Claim / Depends On / Vulnerability columns (per RESEARCH.md Pattern 2)
   - Vulnerability assessment: decided depending on assumed = HIGH vulnerability
   - Key principle from deliberation: "What else are you committed to by asserting this?" (Brandom's inferential probe)
   - Examples from deliberation lines 150-158 (Windows support, retry threshold, cross-phase corpus)

7. **Auto-Progression Rules** — Type-based rules replacing [grounded] gate (per RESEARCH.md Pattern 5):
   | Type | Rule |
   |------|------|
   | evidenced | Auto-ready if verification >= cited |
   | decided | Auto-ready (decisions are resolved) |
   | assumed | Auto-ready ONLY if verification >= reasoned |
   | open | NEVER auto-ready (requires research) |
   | projected | Auto-ready if projected phase exists in ROADMAP.md |
   | stipulated | Auto-ready (choices are resolved) |
   | governing | Auto-ready (framework commitments) |

Do NOT include examples drawn from this project's own CONTEXT.md files (those are in the deliberation for reference but would be confusing in a generic reference doc). Use the generic examples from the deliberation tables instead.
