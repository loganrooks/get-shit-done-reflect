**New Step: Branch Protection Bypass Detection (CI-05)**

Add a detection step that checks recent commits on main for missing or failed required status checks:

1. Query required status checks:
   ```bash
   gh api "repos/$REPO/branches/main/protection/required_status_checks" \
     --jq '.contexts[]' 2>/dev/null
   ```
   If this returns an error (no branch protection configured), skip bypass detection and note in output.

2. Get the 5 most recent commits on main:
   ```bash
   gh api "repos/$REPO/commits?sha=main&per_page=5" \
     --jq '.[].sha' 2>/dev/null
   ```

3. For each commit, check if required checks passed:
   ```bash
   gh api "repos/$REPO/commits/$SHA/check-runs" \
     --jq '[.check_runs[] | {name: .name, conclusion: .conclusion}]' 2>/dev/null
   ```

4. If any required check is missing or has conclusion != "success", create a signal:
   - `summary`: "Branch protection bypassed: commit {sha} on main missing/failed required check '{check_name}'"
   - `signal_type`: "deviation"
   - `signal_category`: "negative"
   - `severity`: "critical" (bypassed protection is always critical)
   - `tags`: ["ci", "branch-protection", "bypass"]
   - `confidence`: "high"
   - `confidence_basis`: "Direct API query of commit check-runs vs required status checks"

**Important:** Limit to 5 commits to avoid rate limiting (research Pitfall 2). Use `2>/dev/null` on all gh api calls to handle permission errors gracefully.

**New Step: Test Regression Detection (CI-06)**

Add a detection step that compares test counts between the two most recent successful CI runs:

1. Get the 2 most recent completed runs for the CI workflow:
   ```bash
   gh run list --workflow "CI" --limit 2 --status completed \
     --json databaseId,conclusion --jq '.[].databaseId'
   ```

2. For each run, attempt to extract test count from logs:
   ```bash
   gh run view "$RUN_ID" --log 2>/dev/null | grep -oE '[0-9]+ passed' | head -1 | grep -oE '[0-9]+'
   ```

3. If both counts extracted and newer < older, create a signal:
   - `summary`: "Test regression detected: test count dropped from {old} to {new} between CI runs"
   - `signal_type`: "deviation"
   - `signal_category`: "negative"
   - `severity`: "notable"
   - `tags`: ["ci", "test-regression"]
   - `confidence`: "low"
   - `confidence_basis`: "Log parsing of vitest output -- format-dependent, may miss non-vitest runners"

4. If log parsing fails for either run, skip regression detection silently (do NOT report false data). Document this as a blind spot.

**Important:** Test regression detection is LOW confidence per research findings. The sensor should treat extraction failure as expected, not exceptional. Add "test count parsing is format-dependent" to the blind_spots section.

**Create `hooks/gsd-ci-status.js`:**

Follow the exact pattern of `hooks/gsd-check-update.js` (background spawn, cache file, detached process, unref):

```javascript
#!/usr/bin/env node
// Check CI status in background, write result to cache
// Called by SessionStart hook - runs once per session

const fs = require('fs');
const path = require('path');
const os = require('os');
const { spawn } = require('child_process');

const cacheDir = path.join(os.homedir(), '.claude', 'cache');
const cacheFile = path.join(cacheDir, 'gsd-ci-status.json');

// Ensure cache directory exists
if (!fs.existsSync(cacheDir)) {
  fs.mkdirSync(cacheDir, { recursive: true });
}

// Run check in background
const child = spawn(process.execPath, ['-e', `
  const fs = require('fs');
  const { execSync } = require('child_process');
  const cacheFile = ${JSON.stringify(cacheFile)};

  // Pre-flight: check gh CLI availability and auth
  try { execSync('command -v gh', { stdio: 'ignore' }); }
  catch { process.exit(0); } // Silent exit -- no gh CLI

  try { execSync('gh auth status', { stdio: 'ignore', timeout: 5000 }); }
  catch { process.exit(0); } // Silent exit -- not authenticated

  // Get current branch
  let branch = 'main';
  try { branch = execSync('git branch --show-current', { encoding: 'utf8', timeout: 5000 }).trim(); }
  catch {}

  // Get latest run for current branch
  let runs = [];
  try {
    runs = JSON.parse(execSync(
      'gh run list --branch "' + branch + '" --limit 1 --json conclusion,displayTitle,name,createdAt,headSha,status',
      { encoding: 'utf8', timeout: 10000 }
    ));
  } catch {}

  const result = {
    branch,
    latest_run: runs[0] || null,
    checked: Math.floor(Date.now() / 1000)
  };

  fs.writeFileSync(cacheFile, JSON.stringify(result));
`], {
  stdio: 'ignore',
  windowsHide: true,
  detached: true
});

child.unref();
```

Key design decisions:
- **Background spawn** (not synchronous): session start is NOT blocked by gh API latency
- **Silent degradation**: if gh is missing or unauthenticated, process.exit(0) with no output
- **Cache file**: `~/.claude/cache/gsd-ci-status.json` -- read by statusline
- **Timeout on all execSync**: 5s for auth, 10s for run list -- prevents hangs

```javascript
// CI status from SessionStart hook cache
let ciStatus = '';
const ciCacheFile = path.join(homeDir, '.claude', 'cache', 'gsd-ci-status.json');
if (fs.existsSync(ciCacheFile)) {
  try {
    const ciCache = JSON.parse(fs.readFileSync(ciCacheFile, 'utf8'));
    // Only show if cache is less than 1 hour old
    const age = Math.floor(Date.now() / 1000) - (ciCache.checked || 0);
    if (age < 3600 && ciCache.latest_run && ciCache.latest_run.conclusion === 'failure') {
      ciStatus = '\x1b[31mCI FAIL\x1b[0m | ';
    }
  } catch (e) {}
}
```
