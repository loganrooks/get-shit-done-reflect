Fix the Gemini CLI installer to preserve MCP tool references and tool_permissions content instead of stripping them. Add `convertClaudeToGeminiAgent` to module exports and add unit + integration tests.

Purpose: Gemini CLI now supports MCP servers (verified in prior research). The installer should not strip MCP tool references from agent frontmatter during conversion. This single fix also addresses tool_permissions preservation because the same code path handles allowed-tools arrays containing MCP tools.

Output: Fixed `bin/install.js` with passing tests covering the corrected behavior.
</objective>
