## Amendment 2: Deprecate Gemini CLI & OpenCode

**Decision document:** `.planning/deliberations/drop-gemini-opencode-focus-codex.md`

**Changes to implement:**

1. **capability-matrix.md:** Add deprecation notice for Gemini CLI and OpenCode columns. Mark as "community-maintained, not tested by GSD Reflect team."

2. **README/install docs:** Note that `--gemini` and `--opencode` flags still work but are unsupported. `--claude` and `--codex` are the supported runtimes.

3. **New workflows:** Stop adding `<capability_check>` blocks for Gemini/OpenCode. Binary Claude/Codex branching only.

4. **ROADMAP.md:** Update Phase 60 description — no longer "Cross-Runtime Adaptation" (which implied all 4 runtimes). Rename to "Sensor Pipeline & Codex Parity" or similar.

5. **REQUIREMENTS.md:**
   - XRT-01: Narrow from "every hook-dependent feature specifies Codex CLI degradation path" (already Codex-specific) — no change needed
   - XRT-02: Update to reference Codex only, not "cross-runtime" generically
   - SENS-02/SENS-03: Already Codex-specific, no change needed

## Amendment 3: New Requirements

Add to REQUIREMENTS.md under a new **Codex Substrate** section:

```markdown
### Codex Substrate

- [ ] **CODEX-01**: Runtime capability resolver (`automation.cjs`) correctly identifies `codex-cli` capabilities — task_tool, hooks status (under development), sandbox modes. Heuristic fallback does not default to "constrained" for Codex.
  - *Motivation:* GPT-5.4 audit confirmed `resolve-level --runtime codex-cli` returns capped result due to code only recognizing `claude-code` and `full`
  - *Dependencies:* None

- [ ] **CODEX-02**: Agent and sensor discovery works across `.md` (Claude Code) and `.toml` (Codex) formats, in both `.claude/` and `.codex/` paths. `sensors list`, `checkAgentsInstalled()`, and workflow sensor auto-discovery all handle both formats.
  - *Motivation:* Claude audit G2 (sensor glob), GPT-5.4 audit §2-3 (agent verification), confirmed with proof commands
  - *Dependencies:* None

- [ ] **CODEX-05**: `cross-runtime-parity-research.md` exists as a living document recording last-audited Codex CLI version, validation commands used, and expected artifact shapes. Post-install smoke test validates Codex artifacts against this document.
  - *Motivation:* GPT-5.4 drift audit §Monitoring: "no durable record of what version was last audited"
  - *Dependencies:* None
```

**Requirement numbering note:** CODEX-03 (adapter blocks) and CODEX-04 (behavioral guardrails) reserved for v1.21 when the adapter layer is designed with more deployment experience.
