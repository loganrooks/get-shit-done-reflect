  <files>get-shit-done/bin/lib/sensors.cjs</files>
  <action>
Update 5 locations in sensors.cjs to use namespace-aware regex matching both `gsd-` and `gsdr-` prefixes:

1. **Line 24** (cmdSensorsList discovery filter):
   Change `/^gsd-.*-sensor\.md$/.test(f)` to `/^gsdr?-.*-sensor\.md$/.test(f)`

2. **Line 50** (cmdSensorsList name extraction):
   Change `file.replace(/^gsd-/, '')` to `file.replace(/^gsdr?-/, '')`

3. **Line 112** (cmdSensorsBlindSpots pattern for --name filter):
   The hardcoded `'gsd-' + sensorName + '-sensor.md'` must try both prefixes.
   Replace the exact-match pattern approach with a regex filter. Change:
   ```
   const pattern = sensorName
     ? 'gsd-' + sensorName + '-sensor.md'
     : null;
   ```
   to:
   ```
   const namePattern = sensorName
     ? new RegExp('^gsdr?-' + sensorName.replace(/[.*+?^${}()|[\]\\]/g, '\\$&') + '-sensor\\.md$')
     : null;
   ```
   And update the filter on line 118 from `if (pattern && f !== pattern) return false;`
   to `if (namePattern && !namePattern.test(f)) return false;`

4. **Line 117** (cmdSensorsBlindSpots discovery filter):
   Change `/^gsd-.*-sensor\.md$/.test(f)` to `/^gsdr?-.*-sensor\.md$/.test(f)`

5. **Line 132** (cmdSensorsBlindSpots name extraction):
   Change `file.replace(/^gsd-/, '')` to `file.replace(/^gsdr?-/, '')`

The `r?` quantifier means: zero or one `r`, so `gsd-` matches (no r) and `gsdr-` matches (one r).
  </action>
